# Sequential Voting With Relational Box Fields for Active Object Detection

- 论文：[CVPR 2022 官方页面](https://openaccess.thecvf.com/content/CVPR2022/html/Fu_Sequential_Voting_With_Relational_Box_Fields_for_Active_Object_Detection_CVPR_2022_paper.html)
- 代码与可视化：[SequentialVotingDet 官方项目页](https://fuqichen1998.github.io/SequentialVotingDet/)

## 一句话总结

这项工作不先把“手”和“候选物体”各自检测后再匹配，而是把已检测手框当作搜索起点，让框内每个像素对活动物体框投票，并通过最多五步的序列决策持续修正结果。

## 研究背景与问题

活动物体是正在被手操作的物体。100DOH 一类方案通常先用通用检测器找手和物体，再预测交互关系；困难在于抓握会造成重遮挡，通用检测器可能根本不给出正确物体候选，而手指姿态、掌心位置等本可用于推断物体，却只在后续匹配阶段被利用。作者因此把问题改写成“给定手框，直接回归对应物体框”，避免候选集合成为硬瓶颈。

## 方法总览

输入图像先经改造的 ResNet-101 编码器、全图 self-attention、DeepLabV3+ 的 ASPP 与双线性上采样，得到与原图同分辨率的 256 维特征。一个 `1×1` 卷积策略头输出 14 个通道，组成两套字段：`Hand-to-Object Box Field (F^ho)` 负责从手像素提出第一版物体框，`Object Refinement Box Field (F^oo)` 负责从当前物体框内部继续精修。每次从字段中取框内像素预测，经置信度加权的离散直方图投票得到下一状态；相对位移、宽高变化均低于 `0.05` 时停止。

## 方法详解

`Relational Box Field (RBF)` 在每个位置保存五元组 `(r, θ, h, w, c)`：像素到目标框中心的极坐标偏移、目标宽高及置信度。训练时对重叠框先裁掉交叠区域，像素框参数使用 Smooth-L1，置信度使用 Focal Loss；再对每个实例平均，防止大框凭像素数主导损失。

投票阶段分别建立中心、宽度和高度直方图。像素预测落入某个离散槽时，以 `c` 为权重累加，三个直方图的峰值组合成输出框。这样，手指和可见物体纹理产生的一致预测会压过手腕、背景与遮挡区域的离群值。

序列被定义为 MDP：状态包含当前物体框、固定手框和图像特征；动作是框中心与宽高的四维偏移；转移是确定性的；奖励为更新后框与真值的 GIoU。训练先用 `L_Fho + L_Foo` 做 100 epoch 模仿学习，再冻结图像特征提取器，以 horizon `T=5`、折扣 `γ=1` 优化策略头，并保留两项字段监督作为辅助损失，从而缓解只训练单步预测器后反复调用产生的分布漂移。

## 实验与证据

100DOH 含 99,899 帧、189.6K 手框和 110.1K 活动物体框。使用同一 Faster R-CNN 手框时，本文在手—物体元组上取得 AP75/AP50/AP25=`29.90/53.02/57.15`，高于 HOTR 的 `29.30/49.27/57.80`，其中 AP50 提升 3.75 点；换成真值手框后达到 `40.05/54.82/64.86`。MECCANO 有 64,349 帧但无手框对应关系，作者用 100DOH 手检测器加最近物体规则生成伪配对；在该集重训后 AP50 从 100DOH Detector 的 `20.18` 提升到 `26.25`，约为 30% 相对增益。

消融直接支持两个核心设计。100DOH 上，中心像素、参数平均、加权投票的 AP75 分别为 `25.00/25.59/29.90`；MECCANO 上对应 AP50 为 `22.85/25.76/26.25`。迭代次数方面，100DOH 一次投票 AP50=`49.58`，两次即到 `53.11`；MECCANO 从一次的 `22.61` 持续升至自动收敛的 `26.25`，说明跨域条件下多步修正更重要。

跨数据集结果还区分了“结构泛化”和“重训收益”。直接把 100DOH 模型放到 MECCANO，本文得到 AP75/AP50/AP25=`9.09/16.61/23.97`，而 100DOH Detector 的 AP50 只有 `11.17`；使用自动伪配对重训后，三项上升为 `12.99/26.25/34.88`。这意味着改进并非只来自对目标域标注的拟合。公平性上，100DOH 主表中本文与 Simple Baseline、100DOH Detector 都使用 AP50=`89.59` 的同一 Faster R-CNN 手源，参数量分别约 48M 与 47M，性能差异不能简单归因于更强手检测器或大幅扩容。值得注意的是，AP25 上 HOTR 略高于本文，但本文 AP50、AP75 更强，优势主要体现在框的中高精度定位，而非只找到大致区域。

复现时还要保持论文的两阶段优化顺序：先让两个关系字段学会稳定地产生可投票框，再冻结高成本图像编码器训练序列策略。若从随机策略直接做强化学习，早期动作几乎没有有效奖励，既偏离作者设置，也无法判断失败来自投票表达还是探索不足。

## 对 YOLO-Agent 的启发

可把 YOLO 的手检测结果视作 query，而不是要求检测头先找出所有被遮挡物体。候选模块应只消费共享特征、手框和当前物体框，输出像素级 `(r, θ, h, w, c)`，特别适合“人手遮住大半目标”的细分场景；若只是把普通 box regressor 多跑几次，则没有验证 RBF 的价值。

**对照组**固定同一 YOLO 手检测器、输入尺寸、训练划分、增强、seed 与推理阈值：A 为“检测全部物体后按框中心最近匹配手”，B 为“从手框单次直接回归物体框”，C 为 `F^ho→F^oo` 加权投票但仅监督训练，D 在 C 上增加论文式五步 RL。**指标**同时报告 100DOH tuple AP50/AP75、活动物体 AP50、遮挡程度分桶 AP50、平均迭代数与单手延迟；另做 vote/mean/center 和 1/2/3/自动停止消融。**失败判断**预先写死：D 在三个种子上相对 C 的 AP50 均值提升不足 1 点，或 AP75 下降；C 不优于单次回归 B；遮挡重子集无收益；亦或每手额外延迟超过基础 YOLO 的 25%，任一发生就不进入默认 Agent 流程。

## 优点

- 数据流围绕“手到物体”的因果线索设计，绕开通用物体候选漏检。
- 稠密投票对局部遮挡和背景噪声有明确鲁棒性证据。
- 自动终止允许容易样本少迭代，困难样本继续修正。

## 局限

- 方法假设手框可靠；真值手框与检测手框之间的结果差距显示上游错误会直接传递。
- 每只手都要执行高分辨率字段投票与多步更新，多手场景成本明显高于一次检测。
- MECCANO 训练依赖最近中心伪配对，远距离工具、双手共持和错误手检可能制造系统性噪声。
- 确定性策略对多解抓握表达有限，作者也把随机策略留作后续方向。
