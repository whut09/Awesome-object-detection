# Cut and Learn for Unsupervised Object Detection and Instance Segmentation

## 一句话总结

该工作从 DINO ViT-B/8 最后一层注意力的 key 特征构造 patch 亲和矩阵，以 MaskCut 迭代提取多个实例掩码，再用 DropLoss 训练 Cascade Mask R-CNN，并通过三轮预测回灌把粗糙的图分割结果放大为可跨数据集零样本推理的检测与实例分割模型。

## 研究背景与问题

自监督 ViT 已表现出隐式的感知分组能力，但 DINO 注意力或 TokenCut 通常只找到一幅图中的单个显著对象，无法覆盖真实场景中的多实例；另一方面，FreeSOLO 等无监督实例分割方法往往需要下游域内的无标签数据。论文要解决的是：仅使用约 130 万张无标注 ImageNet 图像，既生成多对象伪标签，又避免漏标实例被检测损失误当成背景，最终训练一个能直接迁移到 COCO、UVO、VOC、绘画和交通场景的类别无关检测器。

这一问题的关键不是把伪标签做得完全准确，而是让后续检测器能够容忍伪标签的低召回率。标准检测损失会惩罚所有未匹配预测；当 MaskCut 漏掉真实物体时，模型若正确发现该物体反而会收到负梯度。因此，伪标签生成和鲁棒学习必须联合设计。

## 方法总览

完整数据流分为三个阶段：首先，MaskCut 对每张 ImageNet 图像执行三次“特征提取—归一化割—前景屏蔽”，得到多个粗掩码，经 CRF 后处理并转换为边界框；其次，使用这些掩码和框训练由 DINO 权重初始化的 Mask R-CNN 或 Cascade Mask R-CNN，并以 DropLoss 忽略疑似漏标区域的训练损失；最后，将检测器高置信预测并入伪标签，连续进行三轮自训练。实际推理阶段只运行训练好的检测器，不再需要 DINO、NCut、MaskCut 或 CRF。

## 方法详解

### MaskCut

图像被缩放至 \(480\times480\)，输入 DINO 预训练的 ViT-B/8。对 patch \(i,j\) 的 key 特征 \(K_i,K_j\) 计算余弦相似度 \(W_{ij}\)，并以 \(\tau_{\text{ncut}}=0.15\) 二值化亲和关系：低于阈值设为 \(10^{-5}\)，否则设为 1。随后求解归一化割的广义特征值问题：

\[
(D-W)x=\lambda Dx
\]

取第二小特征值对应的特征向量，以均值为阈值生成二分掩码。前景选择同时使用两个先验：前景应包含特征向量绝对值最大的 patch；前景不应覆盖四个角中的两个或更多。得到一个实例后，将其节点从亲和矩阵中屏蔽，再重新执行 NCut，而不是直接取第三、第四小特征向量。默认迭代三次，因此一张图最多产生三个初始对象掩码。

### DropLoss

对候选区域 \(r_i\)，令其与全部伪真值的最大交并比为 \(\operatorname{IoU}^{\max}_i\)，损失写为：

\[
L_{\text{drop}}(r_i)=
\mathbf{1}\!\left(\operatorname{IoU}^{\max}_i>\tau_{\text{IoU}}\right)
L_{\text{vanilla}}(r_i)
\]

默认 \(\tau_{\text{IoU}}=0.01\)。因此，几乎不与任何 MaskCut 掩码重叠的预测不会被作为负样本处罚，为检测器探索漏标区域保留空间；与伪标签有明确关联的区域仍使用检测器原有分类、框回归和掩码损失。

### 训练与自训练

初始检测器采用 ResNet-50，在 ImageNet 伪标签上训练 160K 次迭代，批量大小 16，SGD 学习率 0.005，在 80K 时缩小 5 倍，权重衰减 \(5\times10^{-5}\)，动量 0.9。Copy-Paste 增强会把实例随机缩放到原尺寸的 0.3～1.0，以加强小目标学习。

第 \(t\) 轮自训练保留置信度高于 \(0.75-0.5t\) 的预测；若旧伪标签与新预测的 IoU 大于 0.5，则删除旧标签以去重。每轮从上一轮模型权重继续训练 80K 次迭代，学习率 0.01。由于预测伪标签数量已经充足，自训练阶段不再使用 DropLoss。

## 实验与证据

零样本协议要求模型只接触 ImageNet 图像，在下游数据集上既不微调，也不读取无标签训练图像。由于模型没有类别概念，评估采用类别无关的 \(AP^{box}_{50}\) 和 \(AR^{box}_{100}\)，并额外报告实例分割指标。11 个测试集包括 COCO、COCO20K、VOC、LVIS、UVO、Clipart、Comic、Watercolor、KITTI、Objects365 与 OpenImages。

CutLER 在 11 个数据集上的平均 \(AP_{50}\) 为 24.3，FreeSOLO 为 9.0；平均 \(AR_{100}\) 为 35.5，对比 13.4。COCO val2017 上，Cascade 版本取得 \(AP^{box}_{50}=21.9\)、box AP=12.3、\(AP^{mask}_{50}=18.9\)、mask AP=9.2；FreeSOLO 分别为 9.6、4.2、9.4、4.3。VOC 上 \(AP_{50}=36.9\)、AP=20.2，超过此前最佳 LOST 的 19.8 和 6.7。

在稠密标注的 UVO val 上，CutLER 达到 \(AP^{box}_{50}=31.7\)、box AP=16.1、\(AP^{mask}_{50}=22.8\)、mask AP=10.1；FreeSOLO 的对应结果仅为 10.0、3.2、9.5、3.3。其 \(AP^{mask}_{50}\) 还比使用完整 LVIS 标注训练的 SOLO-v2 高 8.0 个点。

组件消融揭示了增益来源：UVO/COCO 的 \(AP^{mask}_{50}\) 从 Base 的 14.6/13.5，提高到 MaskCut 的 19.3/15.8；加入 DropLoss 后为 20.9/16.6；加入 Copy-Paste 后为 21.5/17.7；最终自训练达到 22.8/18.9。MaskCut 分辨率从 240 提升到 480 时，COCO \(AP^{mask}_{50}\) 从 15.1 增至 17.7；DropLoss 阈值从 0.01 提高到 0.1，则由 17.7 降至 14.4。三轮自训练相对一轮把 UVO \(AP^{mask}_{50}\) 从 20.6 提至 22.8，COCO 从 17.7 提至 18.9。

## 对 YOLO-Agent 的启发

### 专属 Harness

YOLO-Agent 可将该方法实现为“伪标签生成代理—训练策略代理—自训练审计代理”：MaskCut 负责发现实例，训练代理根据伪标签覆盖率启用 DropLoss，自训练代理按置信度阈值合并预测，并监测伪标签数量是否从第一轮的 2.58M 增长到第三轮约 3.03M，同时防止重复实例累积。

验证时应固定同一 YOLO 实例分割结构、ImageNet 数据、训练步数和初始化，设置五个严格控制组：A 为 TokenCut 单掩码加标准损失；B 为 MaskCut 加标准损失；C 为 MaskCut 加 DropLoss；D 在 C 上加入缩放 Copy-Paste；E 在 D 上加入三轮自训练。核心指标为 COCO 与 UVO 的类别无关 \(AP^{box}_{50}\)、\(AR^{box}_{100}\)、\(AP^{mask}_{50}\) 和 mask AP，并单独记录每轮伪掩码数量。

可证伪的失败标准是：在相同预算下，C 相比 B 未能同时提高 UVO 与 COCO 的 \(AP^{mask}_{50}\)，或 E 的第三轮未同时超过第一轮的两项 \(AP^{mask}_{50}\)。出现任一情况，即说明 YOLO 的目标性分数、正负样本分配或无锚匹配机制没有复现 DropLoss 的“保护漏标对象”作用，不能仅凭召回率增长宣称迁移成功。

## 优点

- 将单显著对象发现扩展为多实例伪标签生成，流程简单且不依赖语义类别。
- 明确处理无监督伪标签的漏标问题，而非只追求更精细的初始掩码。
- 训练与推理解耦，昂贵的图割仅用于离线制备标签。
- 对 Mask R-CNN、Cascade Mask R-CNN 和 ViTDet 均有效，具有架构可迁移性。
- 零样本、低标注微调和全监督初始化三种场景都得到实验证据支持。

## 局限

- 每图默认提取三个掩码，对极度拥挤场景仍存在先验上限。
- 角点与显著性规则偏向物体中心化图像，可能误判贴边或大面积背景对象。
- NCut 需要构造 patch 级亲和矩阵，对高分辨率和大规模数据的离线成本较高。
- DropLoss 会放弃大量低重叠区域的监督；阈值过大时，COCO \(AP^{mask}_{50}\) 明显下降。
- 类别无关评估无法证明模型获得了语义概念，也可能把未标注区域或物体部件作为独立实例。

## 评分

- **方法创新：9/10**——创新集中在多轮 MaskCut 与漏标感知损失的有效组合。
- **实验可信度：9/10**——覆盖 11 个跨域数据集，并提供零样本、低标注和系统消融。
- **工程可用性：8/10**——检测器推理常规，但伪标签生成与多轮训练成本较高。
- **对 YOLO-Agent 的价值：9/10**——提供了可代理化的伪标签生产、损失控制和自训练闭环。
- **综合评价：9/10**——以较少结构创新解决了无监督多实例定位中最关键的漏标与训练放大问题。
