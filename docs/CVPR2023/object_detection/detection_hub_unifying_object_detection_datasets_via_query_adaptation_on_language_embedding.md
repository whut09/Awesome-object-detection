# Detection Hub: Unifying Object Detection Datasets via Query Adaptation on Language Embedding

## 一句话总结

该工作以 Sparse R-CNN 为基础，把 COCO、Objects365 与 Visual Genome 的类别名称编码为数据集级语言嵌入，再通过 Cross-Attention 将其注入可学习查询，并用这些查询同时生成解码器和 query-based RPN 的动态卷积核，从而让一个共享检测器在联合训练后分别达到 45.3、23.2 和 5.7 AP。

## 研究背景与问题

作者针对的不是简单类别合并，而是多检测数据集之间同时存在的**分类体系差异**与**标注不一致**：例如相近概念可能采用不同名称，而一个数据集标为前景的物体在另一个数据集中可能被当作背景。实验中的朴素方案将三个数据集合并、映射到扩大后的统一标签层，结果 COCO 从 40.6 降至 40.4 AP，Visual Genome 从 4.9 降至 4.3 AP，说明统一类别索引不能消除负迁移。

该方法进一步指出，语言嵌入虽然能把语义相近的类别放入共同空间，却仍不足以处理不同数据集的类别频率、目标尺度和标注密度分布。其关键设计因此不是单纯替换分类器，而是让每个样本依据来源数据集选择类别集合，并据此生成专属查询 \(Q_D\)；同一组网络参数由不同查询调节检测行为，以隔离互相冲突的监督信号。

## 方法总览

完整数据流为：来源数据集的类别名串联为 `CategorySet`，经 `Tokenizer` 与冻结的预训练语言嵌入得到 \(E\)；BERT 建模类别间关系得到增强表示 \(E'\)；共享查询 \(Q\) 对 \(E\) 做 Cross-Attention，再经 MHSA 形成数据集感知查询；查询驱动 RPN 产生初始候选框，并在动态实例交互头中生成两层动态卷积核；最终目标特征与 \(E'\) 在共同空间做区域—词元对齐，而非通过固定维度的 one-hot 分类器输出类别。

## 方法详解

### 类别对齐嵌入

对于数据集 \(D\) 的类别集合 \(\{c_1,\ldots,c_n\}\)，作者将所有类别名连接成文本序列：

\[
E=\operatorname{Embed}(\operatorname{Tokenizer}(\text{CategorySet}))
\]

不同数据集分别生成 \(E_1,\ldots,E_{N_d}\)，但都位于统一语言空间。它相当于自动标签映射器：语义近似类别自然接近，同时保留数据集独有概念，不需要专家手工建立跨数据集类别表。

分类阶段先以 BERT 得到 \(E'\)，再通过 `FC_E` 和 `FC_V` 将语言表示与交互后的目标特征 \(x^{obj*}\) 投影到同一空间：

\[
S=\operatorname{Sigmoid}\bigl(FC_E(E')\cdot FC_V(x^{obj*})\bigr)
\]

监督目标由 one-hot 类别变为 sub-word grounding：属于真实类别名称的词元标为正，其余词元标为负，并以二元形式的区域—词元对齐损失 \(\mathcal L_{\text{align}}\) 优化。检测匹配仍处于端到端集合预测框架，由 \(\mathcal L_{\text{match}}\) 完成预测与标注框的分配。

### 数据集感知查询

共享可学习查询与当前数据集嵌入执行交叉注意力：

\[
Q_D=\operatorname{CrossAttn}(Q,E),\qquad Q_D^*=\operatorname{MHSA}(Q_D)
\]

\(Q_D^*\) 不只是分类提示，还通过线性层生成动态卷积核 \(K_1(Q_D^*)\) 和 \(K_2(Q_D^*)\)，依次处理候选区域特征。轻量 query-based RPN 同样使用适配查询动态生成卷积核，选择得分最高的 \(N\) 个框作为解码器初始候选。因此，RPN 的候选分布和解码器的实例交互都随数据集变化。

训练时从多个数据集中采样批次，并按每张图像的来源选择对应类别集合；作者重采样数据，使每个数据集均接受约 `1×` 日程的训练量，并对 Objects365 做类别再平衡。推理时输入目标数据集的类别名称及其嵌入，选择相应适配路径直接预测，不进行数据集专属微调。

## 实验与证据

核心实验联合使用 COCO（11.8 万图像、80 类）、Objects365（约 74 万图像、365 类）和 Visual Genome（108,077 图像、约 1,600 类）。ResNet-50、`1×` 日程下，独立训练结果为 43.0/22.2/4.8 AP，联合训练提升至 45.3/23.2/5.7 AP，即分别增加 2.3、1.0、0.9。

该趋势跨骨干保持稳定：Swin-T 联合训练相对独立训练由 46.3/25.1/5.6 提升到 49.2/27.2/6.4；Swin-L 则由 52.5/35.1/8.0 提升到 55.4/36.1/8.9。最终 Swin-L 模型使用 80 万张 COCO、Objects365、Visual Genome 图像，取得 COCO 57.0 AP、Objects365 37.2 AP、Visual Genome 8.9 AP 与 14.4 AP50。

受控消融显示，查询使用的数据范围至关重要。`Instance Sampling` 联合训练仅有 14.5/2.2/0.1 AP，跨全部数据集类别的 `Global Sampling` 为 39.8/20.2/5.0，而按来源选择词表的 `Dataset-aware Sampling` 达到 45.3/23.2/5.7。查询数从 100 增至 300 后，联合训练结果由 42.3/20.3/5.0 提升至 45.3/23.2/5.7。

组件演化实验中，朴素合并为 40.4/17.4/4.3；加入 Category-aligned Embedding 后达到 43.1/21.3/5.2；再加入解码器查询适配为 44.0/21.9/5.4；最后加入 RPN 查询适配后达到 45.6/23.5/5.7，表明语言分类、解码器适配和候选框适配具有可累积贡献。

在由 Clipart、COCO、Comic、DeepLesion、DOTA、Kitchen、KITTI、LISA、VOC、WaterColor、WiderFace 构成的 UODB 上，统一采用 ResNet-50，并报告各数据集 AP50。方法平均达到 71.0，超过 UniDA 的 64.2 和独立训练基线的 59.4。

## 对 YOLO-Agent 的启发

### 专属 Harness

YOLO-Agent 可将数据集或任务描述视为显式控制变量，而不是只在输出端切换类别表：若其检测器含查询解码器，应分别在候选生成与实例解码处注入语言适配查询；若采用纯密集 YOLO 头，则需验证由语言嵌入生成分类卷积核或通道调制参数能否复现本文“双边界适配”的作用，不能只替换文本分类器。

建议设置四组严格控制实验：A 为 COCO、Objects365、Visual Genome 分别训练；B 为联合数据加手工标签映射及共享分类头；C 为 B 加区域—词元对齐，但不适配检测查询或卷积核；D 为 C 加来源数据集感知的候选生成与解码适配。统一报告三个数据集的 AP、COCO 的 AP50/AP75/APS/APM/APL，以及 UODB 十一数据集 AP50 和平均值。

可证伪标准是：若 D 相对 C 的三数据集平均 AP 增益不大于 0，或 D 在任一数据集上低于 A，则不能声称数据集感知适配产生了联合训练协同；若 C 已达到 D 的结果，则说明收益来自语言分类器，而非本文强调的 query adaptation。还应加入 `Global Sampling` 对照；若其不弱于按来源选择词表，数据集隔离假设同样不成立。

## 优点

- 将标签体系统一与数据分布适配拆成两个明确问题，避免把语言嵌入当作万能标签映射。
- 适配同时覆盖 query-based RPN 和解码器，而非仅修改最终分类层。
- 联合训练后直接在各数据集评测，不依赖先预训练、再逐数据集微调的流程。
- 在常规三数据集和跨医学、航拍、漫画、人脸等领域的 UODB 上均给出一致证据。
- 消融覆盖采样范围、查询数量、组件演化、骨干网络与跨数据集泛化。

## 局限

方法仍要求推理前指定目标数据集及其类别词表，因此更接近多域统一检测器，而非无需词表的开放世界检测器。Visual Genome 约 1,600 类，其性能会受到语言序列最大长度限制，作者也将此列为主要瓶颈。BERT、交叉注意力和两处动态核生成增加计算与部署复杂度；类别名称的歧义、同名异义以及细粒度视觉差异也未被语言相似性自动解决。

## 评分

- **方法新颖性：8.5/10**——语言类别对齐与检测查询适配形成了紧密的数据流闭环。
- **实验可信度：9/10**——包含三套大型检测数据、十一域 UODB 和多项可解释消融。
- **工程可迁移性：7.5/10**——适合查询式检测器，但迁移到纯密集 YOLO 头需要重新设计动态调制位置。
- **开放性与复现性：6.5/10**——训练配置较清楚，但论文未给出官方代码仓库。
- **综合评分：8.3/10**——其核心价值是证明多数据集联合检测必须让检测行为本身感知数据集，而不能停留在标签空间合并。
