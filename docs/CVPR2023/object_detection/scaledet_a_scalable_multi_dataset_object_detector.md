# ScaleDet: A Scalable Multi-Dataset Object Detector

## 一句话总结

该工作将 LVIS、COCO、Objects365 与 OpenImages 的标签以不合并的离散并集组成 2249 类语义空间，再用冻结的 CLIP/OpenCLIP 文本嵌入、硬标签损失 \(L_{hl}\) 和软语义损失 \(L_{sl}\) 训练一个 CenterNet2 检测器，使模型可通过替换测试标签文本嵌入，在已见或未见数据集上直接检测或微调迁移。

## 研究背景与问题

多数据集检测的关键矛盾不是图像数量不足，而是标签定义不一致：OpenImages 的“footwear”与 Objects365 的“sneakers”语义接近，却不能简单视为同一类别；“boy”与“girl”同样相近，但合并会破坏判别边界。人工重标注、WordNet 合并或为每个数据集维护独立分类器，都会随数据集数量增加而扩大人工成本或模型复杂度。

作者要求模型同时满足三点：容纳不同规模和长尾程度的数据集；联合训练后在每个上游数据集上不弱于单数据集模型；无需重构分类头即可迁移到具有新领域或新类别的下游数据集。为此，问题被限定为重写两阶段检测器的分类监督，而保留候选框生成和边界框回归路径。

## 方法总览

模型建立在 CenterNet2 两阶段检测器上。训练批次从多个数据集合并后的样本池随机取得，经图像骨干与候选区域模块产生区域视觉特征 \(v_i\)。所有数据集的类别名称通过提示模板送入冻结的 CLIP 或 OpenCLIP 文本编码器，得到文本特征 \(t_j\)。

分类不再使用固定可学习权重，而是计算区域与全部文本特征的余弦相似度。\(L_{hl}\) 将区域绑定到其标注类别，\(L_{sl}\) 则要求完整相似度向量逼近类别间的文本语义关系。推理时保留检测器，只把训练期统一词表替换成目标数据集的文本嵌入；若类别未在训练集中出现，该边界即构成零样本开放词汇检测。

## 方法详解

### 统一语义标签空间

对于 \(K\) 个标签集 \(L_1,\ldots,L_K\)，方法采用离散并集：

\[
L=L_1\coprod L_2\coprod\cdots\coprod L_K
\]

同名或近义标签仍保留各自的数据集身份，因此不会因错误合并传播监督噪声。四个上游数据集共含 2249 个标签和约 370 万张训练图像。

每个标签使用多种类似“a photo of a person”的提示，其文本嵌入取平均。标签 \(l_i,l_j\) 的余弦相似度按第 \(i\) 行的最小值与最大值归一化到 \([0,1]\)，形成 \(n\times n\) 的语义矩阵 \(S\)。标签拼接、提示编码和矩阵 \(S\) 均离线计算，不增加检测器训练或推理阶段的额外网络分支。

### 视觉—文本对齐损失

区域特征与全部标签文本特征的匹配向量为：

\[
\mathbf c_i=[\cos(v_i,t_1),\ldots,\cos(v_i,t_n)]
\]

硬标签分配使用带温度 \(\tau\) 的 sigmoid 与二元交叉熵：

\[
L_{hl}=\operatorname{BCE}\bigl(\sigma(\mathbf c_i/\tau),l_i\bigr)
\]

它在概率空间中拉开类别边界。软标签分配则使用：

\[
L_{sl}=\operatorname{MSE}(\mathbf c_i,\mathbf s_i)
\]

其中 \(\mathbf s_i\) 是语义矩阵中真实类别对应的一行。该项并不要求近义类别完全相同，而是让视觉相似度保持文本语义的连续结构。最终分类监督及总目标为：

\[
L_{lang}=L_{hl}+\lambda L_{sl},\qquad
L_{\text{ScaleDet}}=L_{bbox}+L_{lang}
\]

训练保留原检测器的框回归和候选区域相关损失；测试只替换词表文本特征，并可选择直接迁移或在目标集上微调。

## 实验与证据

上游评测使用 COCO 80 类、LVIS 1203 类、Objects365 365 类和 OpenImages V6 601 类。前三者报告 IoU 0.5:0.95 的 mAP；OpenImages 按官方层级标签协议报告 mAP@0.5。下游采用 ODinW 的 13 个数据集，分别测试直接迁移与微调迁移。

在 ResNet50、CLIP 文本特征和 640×640 输入的受控实验中，训练集从 LVIS 扩展到四个数据集后，四个上游集的平均 mAP 从单集基线水平提升至 46.0；LVIS/COCO/Objects365/OpenImages 分别达到 36.8/47.1/30.6/69.4。与相同 COCO+Objects365+OpenImages 设置的 UniDet 相比，平均 mAP 为 47.7 对 45.4；联合训练相对单集训练的增益为 2.6 对 1.1。

与 Detic 的同条件 LVIS+COCO 对照中，平均 mAP 为 39.1 对 38.4。即使 Detic 使用 1260 万张图像和约 2.2 万类，而四数据集版本仅使用约 370 万张检测图像和 2249 个标签，后者仍取得 41.9 对 38.9。

Swin-Base 配合 OpenCLIP 时，上游平均 mAP 为 56.5；分别在目标集微调后达到平均 58.0。论文汇总的各数据集最佳结果为 LVIS 50.7、COCO 58.8、Objects365 46.8、OpenImages 76.2，这些是不同直接评测或微调配置下的最佳值，并非同一行共享检查点。

ODinW 上，ScaleDet-B 的直接迁移为 47.3，微调迁移为 71.8；后者超过 GLIPv2-B 的 69.4，但直接迁移低于其 54.2。消融中，四数据集训练仅使用 \(L_{hl}\) 时，ODinW 直接迁移 mAP/mAPC 为 37.9/39.8；加入 \(L_{sl}\) 后升至 39.4/40.9，证明软语义监督的主要收益出现在跨数据集泛化，而非上游拟合。

## 对 YOLO-Agent 的启发

### 专属 Harness

可将 YOLO-Agent 的类别头改造成区域或密集位置特征与冻结文本原型的余弦分类器，同时让 Agent 管理数据集词表拼接、提示生成、语义矩阵缓存及测试词表切换。关键不是让 Agent 自动合并近义词，而是保留标签身份，并以连续相似度提供跨集正则。

建议设置三个严格控制组：A 为每数据集独立 YOLO 分类头；B 为统一离散词表且仅训练 \(L_{hl}\)；C 为同一初始化、采样器、增强、分辨率和训练步数下的 \(L_{hl}+\lambda L_{sl}\)。上游报告四数据集逐集 mAP及其均值，下游报告 ODinW 13 集平均 mAP、按类别平均 mAPC、未见类别 AP，并记录推理延迟与峰值显存。

可证伪标准是：若 C 在至少三次随机种子中，相对 B 的 ODinW 直接迁移平均增益不足 1.0 mAP，或虽然提高下游指标却使任一上游集下降超过 1.0 mAP，或词表扩展后推理成本随标签数产生不可接受的线性增长，则该语义软监督不应作为 YOLO-Agent 的默认训练策略。

## 优点

- 以离散并集代替人工映射，数据集扩展时无需修改多头分类器结构。
- 将语义关系编码成离线矩阵，训练目标简单且没有额外推理网络。
- 同时覆盖上游联合训练、未见类别零样本检测和下游微调迁移。
- 与 UniDet、Detic 的同条件对比及损失消融共同支持核心结论。
- 仅使用检测标注数据，即可达到接近大规模视觉—语言预训练模型的结果。

## 局限

- 直接零样本迁移仍弱于使用海量图文数据的 GLIPv2-B，说明文本原型不能完全替代区域—短语预训练。
- \(n\times n\) 语义矩阵及全词表相似度计算在超大类别空间下仍有存储和分类计算压力。
- 标签语义完全依赖文本编码器与提示模板，细粒度类别、同名异义词和数据集特有定义可能被错误关联。
- 数据集直接混合并采用 repeat factor sampling，尚未系统解决标注密度、负类缺失和领域采样失衡。
- 论文未给出类别数量继续扩大到数万级检测标签时的完整伸缩实验。

## 评分

**8.8/10。** 方法以较小改动解决了多数据集检测中最难扩展的标签空间问题，实验覆盖上游、零样本和微调迁移，受控比较也较充分；扣分主要来自直接开放词汇能力仍依赖文本编码质量，以及大词表计算与标签歧义尚未彻底解决。
