# STDLens: Model Hijacking-Resilient Federated Learning for Object Detection

## 一句话总结

该工作针对联邦目标检测中的 Class-Poison、BBox-Poison 与 Objn-Poison，利用输出层梯度的空间主成分、逐客户端时间不相似度和 σ-density 不确定区间组成三层取证流程，在不预知攻击类型的情况下识别并驱逐恶意客户端，同时恢复受害类别的检测精度。

## 研究背景与问题

论文研究的不是推理阶段贴触发器的传统后门，而是训练阶段的“感知投毒”：攻击者仅控制少量客户端的本地标注，不修改联邦软件或服务器。对每个目标，标注由 `CLASS`、`BBOX`、`OBJN` 三部分组成；攻击者分别可将源类别改为目标类别、把框缩小并移动，或把目标存在性改成不存在。由污染数据训练出的本地模型上传 Trojaned gradients，经 FedAvg 聚合后逐步劫持全局检测器，但非受害类别仍可能保持正常，因此常规整体精度监控难以及时报警。

现有 AUROR 式空间聚类通常把较小簇视为恶意簇，Spectral Signature 则基于 SVD 清除偏离主奇异方向的更新。然而每轮只随机抽取部分客户端，某一轮中恶意客户端未必是少数；良性更新也可能落在两簇边界。K-Means、层次聚类和谱聚类因而会产生不同划分，造成大量误杀，并在移除部分客户端后进一步改变后续梯度分布。

## 方法总览

服务器按预设窗口——实验中每十轮一次——累积参与客户端的模型更新。第一层从检测器输出层提取梯度，以协方差最大特征值对应的两个 Spatial Signature Components（SSC）投影并聚类；第二层按客户端身份串联多个轮次的 SSC，计算 Temporal Signature；第三层通过 σ-density 划出高置信区与不确定区，只立即撤销高置信恶意客户端，把边界客户端放入 watchlist。该机制只存在于联邦训练服务器侧；部署推理时仍使用普通 Faster R-CNN，不增加推理模块或推理开销。

## 方法详解

### 空间签名分析

论文将诚实更新分布记为 \(H\)，投毒更新分布记为 \(P\)，恶意比例为 \(m<1/2\)，混合分布写为：

\[
G=(1-m)H+mP
\]

当两类均值距离相对于 \(m\) 归一化协方差足够大时，二者满足论文定义的 `m-separable robust statistics`。由于反向传播中的梯度衰减，作者只分析信号最明显的输出层梯度，并取协方差最大的两个 SSC 形成二维空间签名。正常类别通常形成单一稠密簇，受到感知投毒的源类别则倾向产生两个分布。

### 时间签名分析

对客户端 \(P_i\)，服务器保存其在窗口内、属于同一空间簇 \(c\) 的签名序列 \(\mathcal{G}_i^c\)。时间签名 \(\Upsilon_\omega\) 计算相邻窗口内签名的平均不相似度，并在各簇结果中取最小值；默认 `DisSim` 使用计算成本较低的 L1 距离，实验示意采用 \(\omega=1\)。良性客户端受随机样本和训练动态影响，跨轮更新更分散；持续追求同一劫持目标的恶意客户端更新更相似，因此时间签名更小。系统比较两个空间簇的平均时间签名，把较小者标记为可疑簇。

这一设计没有替换 Faster R-CNN 的训练损失，也没有增加新的检测头损失。攻击发生在本地监督信号：Class-Poison 修改分类目标，BBox-Poison 修改回归目标，Objn-Poison 修改目标存在性；STDLens观察这些监督变化最终造成的上传梯度分布，而不是访问客户端原始数据或检测损失明细。

### σ-density 不确定性检查

系统在最大特征值对应的 SSC 上分别估计两个簇的均值和标准差，并用正态分布经验法则把 99% 置信要求映射为约 \(3\sigma\) 区间。两个高置信区之间被定义为 σ-uncertain zone：落入其中的客户端不立即删除，而是进入 watchlist；多次进入不确定区才会被移除。时间签名也执行同类置信检查。只有位于低时间不相似度簇、且处于高置信恶意区域的客户端，才被撤销后续联邦训练资格。

## 实验与证据

实验采用 PASCAL VOC 与 INRIA Person，设置 100 个客户端、200 个联邦轮次，每轮随机选择 10% 客户端，使用 FedAvg；检测器为 VGG16 主干的 Faster R-CNN。数据集被等分给各客户端，默认恶意比例为 20%。VOC 的受害源类别为 person，Class-Poison 目标类别为 potted plant；BBox-Poison 将框缩小至原尺度的 0.10。核心指标是受害源类别平均精度 `APsrc`。

在 VOC 的 20% 恶意客户端设置下，BBox-Poison 将 `APsrc` 从 52.43% 降至 18.06%，STDLens 恢复到 52.19%；Class-Poison 从 38.22% 恢复到 52.22%，Objn-Poison 从 37.61% 恢复到 52.29%。INRIA 上 BBox-Poison 从良性 63.49% 降至 28.73%，防御后为 61.71%。三种 VOC 攻击的恶意客户端分别在第 30、20、30 轮被全部清除，且没有误杀良性客户端。

对照实验中，三种方法最终召回率均达到 1.00，但差异集中在精确率与清除速度。Class-Poison 下，STDLens、空间聚类和 Spectral Signature 分别得到 `1.00@30`、`0.95@30`、`0.53@40`；BBox-Poison 为 `1.00@20`、`0.71@30`、`0.26@80`。运行时间分别为 452 ms、342 ms 和 1423 ms，其中 STDLens 的时间分析与 σ-density 检查分别占 84 ms 和 44 ms。

自适应实验进一步验证时间维度的必要性。β-adaptive 攻击让恶意客户端以 5%–15% 概率临时上传正常更新；九组实验中，无防御最低降至 22.36%，STDLens 均恢复到至少 51.56%。γ-adaptive 仅污染本地 80% 或 60% 样本时，Class-Poison 分别将 `APsrc` 降至 40.03% 和 46.01%，防御后恢复至 52.09% 和 51.54%。若攻击延迟到第 100 轮启动，系统从第 101–110 轮窗口发现异常，并在第 130 轮清除全部恶意客户端。

## 对 YOLO-Agent 的启发

### 专属 Harness

YOLO-Agent 若承担多节点持续训练与模型版本发布，可将 STDLens 实现为聚合器前置的“更新审计代理”：从 YOLO 检测头中分别提取类别、框回归和 objectness 相关参数更新，按类别构建 SSC，并跨轮维护客户端时间轨迹。它不应直接复用 Faster R-CNN 的层索引，而应验证 YOLO 多尺度检测头的梯度是否仍具有论文所依赖的双分布与低时间不相似度特征。

建议设置四个严格控制组：`FedAvg 无防御`、`仅空间聚类`、`Spectral Signature`、`完整 STDLens`；统一采用 100 客户端、每轮抽取 10%、20% 恶意客户端和 10 轮审计窗口，分别运行类别替换、框缩放至 0.10、objectness 删除及 β/γ 自适应攻击。指标必须同时报告 `APsrc`、非受害类别 AP、最大召回轮次上的防御精确率、误杀数、恶意客户端全部清除轮次与服务器审计耗时。

可证伪标准是：若完整方案在任一基础攻击下无法于第 30 轮前达到恶意客户端召回率 1.00，防御精确率低于 1.00，或训练结束后的 `APsrc` 比同配置良性 YOLO 低超过 1 个百分点，则不能声称该方法已成功迁移到 YOLO-Agent；若 β/γ 攻击使其表现不再显著优于仅空间聚类，也说明时间签名假设在 YOLO 梯度结构上失效。

## 优点

- 将客户端身份与跨轮轨迹纳入分析，纠正了“较小空间簇必然恶意”的脆弱假设。
- 同时覆盖类别、定位和目标存在性三类劫持，不依赖预先生成的 Trojan 样本训练分类器。
- 明确区分高置信恶意区与不确定区，以延迟决策换取零误杀。
- 防御位于训练服务器，不访问客户端原始数据，也不改变部署模型的推理路径。
- 对间歇伪装、部分样本投毒和延迟启动三类自适应攻击均给出控制实验。

## 局限

- 主要实验固定为 IID 式等量随机分区；强 non-IID 数据可能让良性客户端也呈现稳定且分离的时间轨迹。
- 理论与实现依赖恶意比例 \(m<1/2\)，未覆盖多数客户端串谋或 Sybil 主导的轮次。
- σ-density 使用近似正态分布和固定 99% 置信要求，其跨数据集、跨检测头稳定性尚未充分验证。
- 实验只使用 Faster R-CNN/VGG16，模型无关性主要来自攻击假设，而非多种检测器的实证。
- 驱逐客户端属于不可逆操作；若客户端数据分布稀有，误判的业务代价可能高于常规精度指标所体现的损失。

## 评分

**8.7/10。** 论文准确抓住联邦目标检测中特有的细粒度劫持风险，并通过“空间分离—时间归因—不确定性控制”形成逻辑完整、实验可证伪的防御链。其最有价值的贡献不是新的聚类器，而是证明单轮空间异常不足以支持客户端撤销决策；主要扣分点在于检测器与数据异质性验证范围有限。
