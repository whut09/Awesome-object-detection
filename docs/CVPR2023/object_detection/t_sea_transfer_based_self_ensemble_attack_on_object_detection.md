# T-SEA: Transfer-Based Self-Ensemble Attack on Object Detection

## 一句话总结

该工作仅借助一个冻结的白盒检测器，通过 **Constrained Data Augmentation、Model ShakeDrop 与 Patch Cutout** 分别扩展数据、模型和补丁视角，在不查询目标模型的条件下训练通用人物对抗补丁，并显著降低多个异构黑盒检测器的 AP。

## 研究背景与问题

论文研究的不是逐图生成扰动，也不是通过反复访问目标检测器进行查询攻击，而是训练一个可贴到人物框中心的通用补丁：训练时只能读取单个白盒检测器的梯度，测试时则将同一补丁直接部署到 YOLO v2、YOLO v3、YOLO v3-tiny、YOLO v4、YOLO v4-tiny、YOLO v5、Faster R-CNN 和 SSD 等模型上，以 AP 越低代表攻击越成功。

传统迁移式检测攻击常集成多个代理模型来提高跨模型泛化，但收集、训练和同时运行同任务的多个检测器代价较高。本文将补丁优化类比为模型训练：训练图像与代理模型构成联合输入分布 \(D(X,F)\)，补丁则是待优化参数；补丁对有限图像或单一网络结构过拟合，正是黑盒迁移能力不足的核心原因。

优化目标是在对抗补丁分布中寻找通用补丁 \(\tau\)，最小化自集成图像、检测器和补丁上的检测破坏损失，并加入补丁总变差正则项 \(J(\tau)\)。论文默认以对抗图像中目标置信度的平均值作为攻击损失，采用 \(L_1\) 形式；优化边界严格限定在补丁，检测器权重始终冻结。

## 方法总览

完整流程分为训练阶段与攻击阶段。训练阶段先对输入图像执行受约束增强，再由原始白盒检测器定位目标类别；随后把补丁变换并贴到每个目标框中心，对补丁随机 Cutout，将合成图像送入经过 ShakeDrop 扰动的白盒模型，最小化人物目标置信度并更新补丁。训练达到最大轮数后冻结补丁。

攻击或推理阶段不再执行梯度计算、模型更新或目标查询，也不需要保留 ShakeDrop；系统只把训练完成的补丁贴到测试人物上，直接评估白盒及未参与训练的黑盒检测器。因而“自集成”是一种训练期正则化，而非推理期多模型投票。

## 方法详解

### Enhanced Baseline

作者首先改进 AdvPatch 的训练策略。原始基于平台期的学习率调度器容易把补丁优化中的鞍点误判为收敛并过早大幅降学习率，因此新调度规则同时检查相邻轮损失差及其相对变化，使学习率衰减更平稳。训练补丁相对目标框的边长比例也由 **0.20 降至 0.15**，迫使补丁学习缩放后仍能保留的全局模式，而不是脆弱的局部纹理。

每个批次的数据流为：白盒检测器在干净图像上产生 `bbox_clean`；Patch Applier 根据这些框生成 \(X_{\text{adv}}\)；同一检测器输出 `conf_adv`；取目标置信度均值作为 loss，再由 Adam、SGD、MIM、BIM 或 PGD 等基础更新器修改 \(\tau\)。训练设置为 INRIA 人物训练集、输入 \(416\times416\)、补丁 \(300\times300\)、批量 8、最多 1000 个 epoch。

### Constrained Data Augmentation

数据自集成采用水平翻转、轻微颜色抖动、随机缩放裁剪和小角度旋转。与任意强增强不同，其约束是生成仍可能出现在自然场景中的人物图像，从有限样本虚拟扩展 \(D(X,F)\) 中的 \(X\)，避免补丁只适配固定颜色、尺度和姿态。

### Model ShakeDrop

模型自集成作用于残差块。前向传播把恒等分支 \(z_l\) 与堆叠层输出 \(F(z_l)\) 随机线性组合：

\[
z_L=z_l+\left(\beta+\alpha-\beta\alpha\right)F(z_l)
\]

其中 \(\beta\) 来自伯努利分布，\(\alpha\) 从连续均匀分布采样；反向传播使用另一个同分布伯努利变量改变梯度路径。实验中 \(e=1\)，故 \(\alpha\sim U(0,2)\)，模型扰动概率 \(\phi_s=0.5\)。这相当于从一个残差检测器虚拟采样大量结构变体，而无须准备多个代理模型。

### Patch Cutout

补丁自集成以概率 \(\phi_c\) 在补丁内部随机选中心，遮盖边长比例为 \(\eta\) 的方形区域。实验采用填充值 \(k=0.5\)、\(\eta=0.4\)、\(\phi_c=0.9\)。它阻止攻击能力集中在少数区域，使不同局部纹理共同承担降低检测置信度的作用；测试时使用完整补丁，不保留遮挡。

## 实验与证据

训练集为 **INRIA Person** 的 614 张图像，测试集为 288 张；跨数据集测试使用 **COCO-person 1684 张**和 **CCTV-person 559 张**图像。评价以干净检测结果作为参照，报告 AP，数值越低攻击越强。每次选择一个检测器作为白盒，其余七个作为零查询黑盒模型。

以 YOLO v2 为白盒时，AdvPatch 的白盒 AP/黑盒平均 AP 为 **5.66/36.46**，T-SEA 降至 **1.73/9.16**；在相同攻击面积下，E-baseline 的黑盒平均 AP 为 16.59，NPAP 为 61.45，AdvCloak 为 59.42。以 YOLO v5 为白盒时，Adam 版 AdvPatch 为 **13.39/48.25**，T-SEA 为 **1.37/19.59**。采用 SGD、MIM、BIM、PGD 时，黑盒平均 AP 也分别由 55.97、33.51、32.93、38.70 降至 15.85、15.43、19.15、18.06，说明收益并不依赖单一优化器。

消融实验中，E-baseline 为 **13.39/48.25**；单独加入数据增强得到 18.47/42.42，说明它可能牺牲白盒拟合却改善迁移；单独加入 ShakeDrop 为 12.54/39.40，单独加入 Patch Cutout 为 6.80/32.54，三者组合达到 **1.37/19.59**。调整学习率与训练尺度的受控比较也把 mAP 从原始 PlateauLR、尺度 0.20 的 **38.89** 降到新调度器、尺度 0.15 的 **24.77**。

跨数据集方面，COCO-person 上 AdvPatch 与 T-SEA 的黑盒平均 AP 为 52.54 和 **38.87**；CCTV-person 上则为 34.08 和 **19.91**。物理验证仅展示了在 iPad 上显示补丁后同时干扰 YOLO v5 与 SSD 的案例，属于可行性演示，而非系统化物理鲁棒性评测。

## 对 YOLO-Agent 的启发

### 专属 Harness

若 YOLO-Agent 包含自动选择代理检测器、补丁训练策略和跨模型验证的能力，可将 T-SEA 实现为明确的四组实验：①原始 AdvPatch；②仅含新学习率调度和 0.15 训练尺度的 E-baseline；③分别加入 CDA、ShakeDrop、Cutout；④完整三模块组合。所有组必须共享 INRIA 划分、\(300\times300\) 补丁、\(416\times416\) 输入、1000 epoch、相同贴片面积与基础更新器。

主实验应固定 YOLO v5 为白盒，并以 YOLO v2/v3/v3-tiny/v4/v4-tiny、Faster R-CNN、SSD 为黑盒，记录白盒 AP、七模型平均 AP、各模型 AP，以及 COCO-person、CCTV-person 的跨数据集黑盒平均 AP。Agent 还应显式记录训练期 ShakeDrop 和 Cutout 的随机种子，防止把随机波动误判为策略收益。

可证伪标准是：在至少三个独立种子下，完整组合若不能把黑盒平均 AP 显著降到单模块最佳结果 **32.54** 以下，或其置信区间与 Patch Cutout 单模块重叠，则“多维自集成产生互补迁移增益”的核心主张未被复现；若白盒 AP 很低但跨数据集 AP 不优于 E-baseline，则说明 Agent 只学到了代理模型过拟合补丁。

## 优点

- 把多代理模型集成转化为单模型内部的随机结构集成，符合无查询、资源受限的攻击设定。
- 数据、模型、补丁三个正则化维度与过拟合来源一一对应，模块职责清晰。
- 覆盖八种检测器和五种基础更新方法，并报告跨数据集与简单物理案例。
- 不仅提升黑盒迁移，在多数设置中也同步改善白盒攻击强度。

## 局限

- ShakeDrop 依赖残差结构，论文明确指出其不能直接迁移到 Transformer 检测器。
- 攻击类别和主要数据均围绕人物，尚不足以证明对多类别、小目标或密集场景同样有效。
- 物理实验只有展示性案例，未控制距离、角度、光照、打印材质和遮挡。
- 评价以干净模型预测作为参照，而非标准人工标注 AP，数值不宜直接与常规检测精度比较。
- 高达 1000 epoch 的补丁训练仍有明显计算成本，且未系统报告训练时间与方差。

## 评分

**8.6/10。** 该方法以较小的工程改动解决了单代理检测器的迁移过拟合问题，实验改进幅度大、控制组充分；扣分主要来自对 Transformer、多类别任务及严格物理场景的验证不足。
