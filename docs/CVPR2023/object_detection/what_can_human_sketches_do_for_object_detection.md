# What Can Human Sketches Do for Object Detection?

## 一句话总结

SketchX 团队在 CVPR 2023 提出一种草图驱动的极弱监督检测框架：冻结 CLIP ViT-B/32 主体，仅学习模态专属提示，将跨类别细粒度草图检索模型转化为 Faster R-CNN 区域原型监督，使检测器在没有目标数据集边界框和图像级类别标签的条件下，根据查询草图定位类别、特定实例乃至物体局部。

## 研究背景与问题

该工作不把草图简单视为类别名称的替代品，而是利用其细粒度表达能力区分“同类中的哪一个”和“物体的哪一部分”。系统分别在 QuickDraw-Extended、Sketchy 上学习类别级与实例级草图—照片对应关系，再零样本迁移至 PASCAL-VOC、MS-COCO 和 SketchyCOCO，因而同时涉及开放词汇检测、跨模态匹配和跨数据集泛化。

其核心训练设定被称为 **EWSOD（Extremely Weakly Supervised Object Detection）**：检测器不读取下游检测数据分布，也不使用下游边界框或图像级类别标签；训练图像由 1～7 张 SBIR 单物体照片随机平铺而成，相应草图构成支持集。测试时则输入真实场景图像与查询草图，直接输出与草图语义或实例特征匹配的区域。

传统检测器的固定分类头只能识别预定义类别；传统 WSOD 又依赖图像级标签，并容易只抓住最判别性的局部。论文的问题因此是：能否用已有草图—照片对替代检测标注，同时保留草图对于姿态、外观、动作和局部结构的表达，并泛化到训练阶段未出现的类别和真实场景？

## 方法总览

框架由两条先后训练、推理时组合的数据流构成：

1. **通用 SBIR 阶段**：为草图和照片分别学习 CLIP 视觉提示，得到冻结的草图编码器 \(F_s\) 与照片编码器 \(F_p\)。
2. **检测器适配阶段**：以 Visual Genome 预训练的 ResNet+FPN 和 RPN 产生 1000 个候选框；RoI 特征经过分类、检测和迭代细化嵌入头，与查询草图原型计算相似度。
3. **推理阶段**：不再需要照片编码器或类别标签；给定场景图和一个草图，检测器将候选区域嵌入与草图支持原型匹配，经细化及 NMS 输出框。

## 方法详解

### 模态专属提示学习

草图提示 \(v_s\) 和照片提示 \(v_p\) 均包含 \(P=3\) 个、维度为 768 的可学习向量，注入 CLIP ViT 第一层；ViT 权重保持冻结：

\[
F_s(x)=F_{\text{CLIP}}(x,v_s),\qquad
F_p(x)=F_{\text{CLIP}}(x,v_p).
\]

类别级 SBIR 使用余弦距离三元组损失，间隔 \(\mu=0.3\)。跨类别细粒度 SBIR 使用同类别内的 hard triplet，并加入类别判别损失：样本特征与 CLIP 文本编码器生成的 “a photo of a [class]” 向量做点积分类。独立提示既缩小 sketch-to-photo 域差异，又避免直接微调 CLIP 导致的泛化坍塌。

### 草图原型检测头

基础结构沿用 WSDDN/OICR 式双流 WSOD。固定类别分类器被 512 维开放集嵌入取代：分类头、检测头和细化头分别产生 \(e_{\text{cls}},e_{\text{det}},e_{\text{ref}}\)。查询草图经 \(F_s\) 编码，与一个可学习背景向量共同组成支持矩阵

\[
S=[e_{\text{bg}},f_s^1,\ldots,f_s^{|C|}]^\top .
\]

各头通过嵌入与 \(S\) 的点积获得区域分数。分类流在类别维做 softmax，检测流在候选区域维做 softmax，二者逐元素相乘并跨区域求和，形成弱监督图像级预测。随后以最高分区域为种子，将 IoU 不小于 0.5 的候选赋予同一伪标签，执行 \(K\) 轮 OICR 式细化。

### 照片特征蒸馏

照片编码器只在训练期提供区域级约束。对候选框 \(r\)，检测器的 RoI 特征 \(f_r\) 与裁剪照片经 \(F_p\) 得到的特征进行 L1 匹配：

\[
\mathcal L_{\text{kd}}
=\left\|f_r-F_p(\operatorname{Crop}(p,r))\right\|_1.
\]

总损失为

\[
\mathcal L_{\text{tot}}
=\mathcal L_{\text{ws}}
+\sum_{k=1}^{K}\mathcal L_{\text{ref}}^k
+\lambda\mathcal L_{\text{kd}},\qquad \lambda=1.
\]

检测训练先冻结 backbone 与 RPN，训练 RoI/FC、双流头和细化头 240k 次迭代；再仅冻结 RPN，联合微调其余模块 80k 次。最终采用 IoU 阈值 0.3 的 NMS。

## 实验与证据

训练数据包括：Sketchy 的 125 类、每类 100 张照片且每张至少配有 5 幅实例草图；QuickDraw-Extended 的 33 万幅草图、20.4 万张照片和 110 类。细粒度测试使用经过类别去重的 1,225 对 SketchyCOCO 样本；类别级测试覆盖 VOC 2007 的 20 类和 MS-COCO 的 56 个重叠类。

类别级检测中，本方法在 VOC 2007 获得 **49.3 AP.5 / 69.4 CorLoc**，在 MS-COCO 获得 **25.9 AP.5 / 70.3 CorLoc**。对应最强全监督草图基线 CoAttOD 分别为 34.6/53.9 和 15.0/71.3；本方法 AP.5 高出 **14.7** 与 **10.9**，但 COCO CorLoc 略低 1.0。

SketchyCOCO 实例级检测达到 **15.0 AP.3、17.1 AP.5、16.3 AP.7**，超过 CoAttOD 的 10.4/12.1/11.7、ICMWSD 的 10.3/11.9/10.8，以及极弱监督 E-ICMWSD 的 8.5/10.2/9.4。SBIR 本身也显示出低数据优势：仅用 50% 训练数据时，跨类别 FG-SBIR 达到 **14.7 Acc.@1、40.1 Acc.@5**，而 CCD 为 9.2/32.2。

消融结果支持各组件的因果作用：以 Selective Search 或 Edge Boxes 替换预训练 RPN，SketchyCOCO AP.5 分别下降 **1.3** 和 **2.6**；移除照片编码器的 L1 蒸馏，AP.5 下降 **4.5**；加入前两轮分类器细化分别提升 **2.4** 和 **1.1**，第三轮继续细化反而下降 **0.2**。

## 对 YOLO-Agent 的启发

### 专属 Harness

可将该方法改造成 YOLO-Agent 的“草图条件检测工具”，但应保留严格对照实验：

- **G0**：标准 YOLO，仅以固定类别 ID 查询。
- **G1**：YOLO 区域特征与冻结 CLIP 草图特征直接匹配，不学习模态提示。
- **G2**：加入独立的 \(v_s/v_p\) 提示和草图原型，但移除照片区域蒸馏。
- **G3**：完整方案，包含独立提示、1～7 物体平铺训练、草图原型及 L1 照片蒸馏。

四组必须使用相同候选框数量、相同 unseen-category 划分和相同 NMS；报告 VOC/COCO 的 AP.5、CorLoc，以及 SketchyCOCO 的 AP.3/AP.5/AP.7，并额外记录草图编码与区域匹配延迟。若 **G3 在 SketchyCOCO AP.5 上不能稳定超过 G2，或移除照片蒸馏未造成可复现下降；同时 G3 在未见类 AP.5 上不优于 G1**，则应判定该方法的核心提示—蒸馏链路对 YOLO-Agent 无有效贡献，而非将收益归因于草图输入本身。

## 优点

- 将草图从检索查询提升为开放集检测接口，统一类别级、实例级和局部级意图。
- 不使用目标检测数据集的框或图像级标签，仍显著超过多种 SOD、WSOD 与 EWSOD 基线。
- 只训练少量 CLIP 提示参数，保留基础模型的零样本泛化。
- 明确区分训练期照片教师与推理期草图查询，部署边界清晰。

## 局限

局部检测仅有人工裁剪草图的定性展示，缺少带部件框的数据集和定量指标。结果依赖 Visual Genome 预训练 RPN，因此“无边界框训练”并不等于系统从未利用框监督；RPN 对微小部件缺乏候选框时，后续原型匹配无法补救。多草图被独立处理，尚不能表达“人在狗左侧”等组合关系。训练画布来自规则平铺，与真实遮挡、尺度和上下文分布仍存在差距。

## 评分

**8.7/10。** 创新点明确，训练协议严格，跨数据集零样本结果有说服力；主要扣分来自对预训练 RPN 的依赖、部件任务缺乏量化，以及尚未验证复杂多对象草图组合。
