# Explaining Object Detectors via Collective Contribution of Pixels

**论文**: [arXiv](https://arxiv.org/abs/2412.00666)  
**官方代码**: [tttt-0814/VX-CODE](https://github.com/tttt-0814/VX-CODE)  
**任务**: 目标检测可解释性 / 像素集体贡献分析

## 一句话总结

VX-CODE 把图像切成 patch，以同时衡量框重合与类别分布一致性的检测奖励为目标，通过自上下文 Shapley 值与 interaction 驱动贪心插入/删除，从而找出必须协同出现才支撑检测结果的区域，而不再只给单像素独立归因。

## 研究背景与问题

检测器的输出不是单一类别分数，而是候选框位置、类别概率和后处理共同作用的结果。D-RISE、ODAM、SSGrad-CAM++ 等方法通常按单个像素或局部梯度评估贡献，容易只高亮手、头部等强响应区域，却漏掉“腿与头共同决定框高”“冲浪板与海面共同触发类别”这类组合证据。论文关注的因此不是一般显著图，而是：哪些 patch 单独作用有限，但组合后会显著改变目标框定位或分类。

作者把完整图像的一次最终检测记为 `(Lx, Px)`。对只保留 patch 子集 `S` 的扰动图 `xS`，检测器产生多个候选 `(L, P)`，奖励取其中最大值：定位项为 `IoU(Lx,L)`，分类项为 `Px` 与 `P` 的余弦相似度，两者相乘。插入问题寻找给定数量下使奖励最大的 patch 集；删除问题寻找移除后使奖励最小的集合，因此两条路径分别回答“最少需要什么证据”和“破坏哪些证据最致命”。

## 方法总览

Visual eXplanation of COllective contributions for object DEtectors（VX-CODE）首先建立 patch 玩家集合 `N`，再在每一步从未选 patch 中挑选 `r` 个，使当前集合的检测奖励最大。该贪心选择可分解为 self-context Shapley value 与 self-context interaction：前者描述集合在自身上下文中的净贡献，后者从联合贡献中扣除所有严格子集贡献，显式保留组合效应。最终按 patch 被插入或删除的先后顺序生成热图。

## 方法详解

初始步直接选择使 `f(B1)-f(∅)` 最大的 `r`-patch 组合；后续步把已选集合 `Bk-1` 作为上下文，选择最大化 `f(Bk-1∪B)` 的新组合。`r=1` 只评估个体贡献，`r=2/3` 会覆盖更多子集与交互，能缓解早期奖励近似持平导致的错误排序。作者还从 sequential coalitional game 角度定义 `π*`-index，解释该顺序为什么可以视为一条逐步形成联盟的归因路径，但明确不声称精确逼近经典 Shapley 值。

直接枚举的代价随 `r` 快速增长。Patch Selection（PS）先按单 patch 奖励筛到前 `m=30` 个，再只在其中组合；Step Restriction（SR）仅在前 `γ=0.1` 比例步骤使用组合搜索，之后退化为逐个选择。实现中缺失 patch 用局部零遮挡，也验证了模糊替代。数据流始终是“扰动图像→原检测器全部候选→与目标检测计算框/类别奖励→选择下一组 patch”，不需要改动检测器参数。

这里的“集体贡献”还需要与把多张单像素显著图简单相加区分。若两个 patch 各自插入时奖励几乎不变，但同时出现后目标候选的 IoU 与类别相似度骤升，它们的联合贡献才会被 interaction 保留下来；反过来，两个 individually strong patch 若表达相同纹理，组合奖励没有继续增加，其冗余不会被误解成更重要证据。`r` 因而不仅是一次选择多少块的速度参数，也决定解释器能观察到多高阶的局部联盟。

插入与删除热图也不应互相替代。插入顺序通常优先呈现足以恢复目标的判别性局部，适合解释类别触发因素；删除顺序会覆盖使原检测失效的更完整结构，更适合检查框范围和上下文依赖。对于同一检测，二者差异本身就是诊断信号：若插入只需目标内部一小块，而删除必须同时移除大面积背景，说明模型可能存在背景辅助依赖。

论文对带方形标记的偏置模型、把狗误判为鸟的天空背景以及自行车框被骑车人牵引的案例进行了专门分析，这些设置比普通正确检测更能检验 collective contribution。解释方法若只能在成功样本上复现物体轮廓，却无法指出诱发错误的跨框区域，就没有完成本文定义的目标。

## 实验与证据

- 主实验在 MS-COCO、PASCAL VOC 上解释 DETR 与 Faster R-CNN，比较 Grad-CAM、Grad-CAM++、D-RISE、SSGrad-CAM、ODAM、SSGrad-CAM++；每组统计 1000 个置信度高于 0.7 的检测实例。
- COCO/DETR 上 SSGrad-CAM++ 的插入、删除、整体 AUC 为 `0.871/0.114/0.757`，VX-CODE `r=1` 为 `0.904/0.053/0.851`，`r=3` 达 `0.909/0.052/0.857`。PASCAL VOC/Faster R-CNN 上 `r=3` 的整体 AUC 为 `0.787`，同样高于全部基线。
- 误分类与误定位各取 200 个 DETR/COCO 失败实例。误分类时 SSGrad-CAM++ 为 `Ins 0.647、Del 0.220`，VX-CODE `r=3` 为 `0.738、0.168`；误定位则达到 `0.787、0.078`，说明它能定位天空、骑车人身体等诱发错误的上下文。
- PS/SR 消融在 RTX A6000 上把单实例解释时间从 `768.126s` 降至 `96.253s`；整体 AUC 从 `0.860` 变为 `0.863`，表明大部分有效组合确实集中在早期步骤。
- 在 Grounding DINO 上对比 VPS，VX-CODE 的 `Ins/Del/OA` 为 `0.962/0.211/0.751`，VPS 为 `0.946/0.300/0.646`。论文也给出 pointing game、偏置模型、随机化 sanity check 和框定位/分类加权奖励的补充证据。

## 对 YOLO-Agent 的启发

可把 VX-CODE 作为故障诊断插件，而非训练组件：对 YOLO 的最终框记录目标类别向量与坐标，复用其 reward，在部署失败样本上产生 insertion/deletion 顺序；若背景 patch 与目标局部反复成组出现，Agent 应优先触发数据偏置检查，而不是直接调整置信度阈值。

**Harness**：对照组设为 D-RISE、单 patch 的 `r=1`、组合搜索 `r=2`，并分别开关 PS、SR。观测 COCO 风格插入 AUC、删除 AUC、OA、单框前向次数、解释耗时，以及对误分类/误定位样本的分层结果。通过阈值建议为：三随机种子下 OA 至少比 D-RISE 高 `0.05`，删除 AUC 至少下降 `0.03`，且启用 PS+SR 后耗时降低不少于 `5×`、OA 下降不超过 `0.01`；若热图只覆盖框内高纹理、失败样本收益消失，或解释一次仍超过在线诊断预算，则判定不适合自动集成。

## 优点

- 奖励同时覆盖定位和类别，不把检测解释退化为分类显著图。
- interaction 能揭示背景—目标、部件—部件的联合依赖，并有失败案例支撑。
- 不修改检测器，适配 DETR、Faster R-CNN 和 Grounding DINO。

## 局限

- `r≥2` 仍需大量前向，PS/SR 是启发式加速而非严格近似保证。
- 结果依赖遮挡方式、patch 粒度及既定 reward；作者也承认最佳奖励设计仍是开放问题。
- 插入/删除 AUC 与人类可理解性并不完全等价，热图不能直接证明因果机制。
