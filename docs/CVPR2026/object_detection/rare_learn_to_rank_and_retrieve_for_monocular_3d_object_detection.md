# RARE: Learn to RAnk and REtrieve for Monocular 3D Object Detection

**论文**: [CVF Open Access](https://openaccess.thecvf.com/content/CVPR2026/html/Park_RARE_Learn_to_RAnk_and_REtrieve_for_Monocular_3D_Object_CVPR_2026_paper.html)  
**代码**: [HyeonjeongPark37/RARE](https://github.com/HyeonjeongPark37/RARE)  
**任务**: 单目三维目标检测与置信度估计

## 一句话总结

RARE 不再让单目三维检测器只回归一个容易“均值坍缩”的盒子，而是为每个 RoI 构造三个候选查询，以点式校准加对式排序学习 3D IoU 质量，再按该置信度从多种合理三维配置中取出第一名。

## 研究背景与问题

单张 RGB 图像无法唯一确定深度、尺寸和朝向。论文用针孔模型说明：KITTI 中焦距约 721.54 像素、车辆平均高度约 1.5 米时，一像素高度误差在 20、30、40 米处会分别造成约 0.36、0.81、1.43 米深度变化。传统单点回归会把多个可行解压成并不真实的条件均值；与此同时，基于分类分数、深度不确定性或绝对 IoU 回归的置信度，对轻微深度和角度误差非常敏感，排序常与真实 3D 定位质量错位。

## 方法总览

输入图像先经 DLA34 和多尺度可变形自注意力编码器形成视觉 token；并行的 2D head 输出中心度、二维尺寸与偏移热图，保留 top-N RoI。每个 RoI 经过 RoI Align 后，一路预测类别，另一路由 MLP 生成含 K=3 个元素的 query set。可变形 Transformer decoder 更新三个查询，各自预测二维中心、深度、三维尺寸、yaw 和置信度，训练时同时执行 Learning to Rank 与 Learning to Retrieve，推理时每个 RoI 只返回置信度最高的候选，且不使用 NMS。

## 方法详解

Learning to Rank 先把候选框与同类别真值的最大 3D IoU 定义为真实质量。点式项以均方误差把 sigmoid 置信度锚定到该质量，解决分数尺度不可解释的问题；对式项随机采样质量不同的检测对，对两个未归一化 logit 使用 logistic ranking loss，强制高质量框排在低质量框之前。两项互补：点式项负责全局校准，对式项强化局部次序，避免只做绝对回归时小定位扰动造成排序翻转。

Learning to Retrieve 对同一 RoI 的 K 个候选置信度做 softmax，得到候选责任权重。3D box multi-hypothesis loss 的第一项用这些权重加权每个候选到真值的盒回归误差，让更可信的假设承担更强拟合责任；第二项约束一个 batch 内各槽位的平均选择概率接近 1/K，防止某个查询长期独占概率而使其他候选坍缩。最终目标为二维热图损失、该多假设三维损失和排序损失的加权和。

这套检索不是先生成大量框再做普通 NMS。类别直接由 RoI 特征预测，同一 RoI 内三个查询共享对象身份，却可以在深度、三维高度、尺寸和朝向上形成不同解；推理只比较组内置信度。训练中排序损失看的是整张图的检测对，检索损失看的是单个对象内部的候选责任，因此前者改善跨对象分数次序，后者负责覆盖投影歧义。论文还指出只用对式项会收敛慢且分数缺少绝对含义，只用点式项则线性相关高、单调排序仍弱，这解释了为何两种监督必须共同存在。

## 实验与证据

论文在 KITTI 上训练并按官方 AP3D|40、APBEV|40 评估，同时把冻结模型迁移到 nuScenes frontal validation，以匹配框深度 MAE 检查跨数据集泛化。KITTI Car test 上，RARE 的 AP3D 为 28.83/19.57/17.38（Easy/Moderate/Hard），超过 MonoDGP 的 26.35/18.72/15.97；Pedestrian 为 14.85/10.79/9.04，Cyclist 为 11.17/5.96/5.28。nuScenes 全距离深度 MAE 为 1.05，优于 MonoCD 的 1.33。

关键消融以 MonoDETR 风格的单候选、深度不确定性分数为基线，AP3D 为 24.93/19.04/16.57。仅排序升至 26.54/21.15/18.20，仅检索为 27.81/20.64/17.56，联合达到 28.58/22.05/19.21。置信度消融中，基线 Pearson/Spearman 为 0.540/0.480；点式加对式后为 0.825/0.754。检索损失从朴素多查询、责任加权，到再加入均匀正则，Moderate AP 依次为 19.85、19.99、20.64，说明“多放几个 query”本身不足以形成互补假设。模型为 32.9M 参数、35.3 ms/A100，精度高于 38.9M、36.5 ms 的 MonoDGP。

训练在两张 A100 上以总 batch 32、Adam 初始学习率 0.001 运行 800 epoch，并采用 HTL 线性 warm-up、MixUp3D 与 DivAlign。编码器和解码器各三层、八头、隐藏维 256，RoI Align 输出 7×7；二维分数低于 0.2 的预测被丢弃。几何多样性分析显示候选间的深度、朝向和 IoU 差异会随距离增加，而物体高度差异相对稳定；在 40 米处三个假设覆盖约 1.35 米深度范围，接近一像素量化推导的 1.43 米，这使“多假设”与成像几何建立了可检验联系。

类别结果也揭示方法适用边界：Pedestrian Easy 的 14.85 并未超过所有单目方法，但 Moderate/Hard 达到 10.79/9.04；Cyclist 的三档相对第二名单目方法提升约 52.2%/39.3%/39.7%。作者据此认为细长、小尺度目标的深度歧义更强，更能受益于候选检索。该证据比只报告 Car 更有说服力，也提醒复现时不能仅用车辆平均指标下结论。

此外，测试集 Car 的 APBEV 为 38.46/26.37/23.46，说明排序检索同时改善鸟瞰定位，而非只改变三维评测中的置信度截断。

## 对 YOLO-Agent 的启发

可以把 RARE 拆成“候选生成器”和“质量排序器”接在 YOLO 三维头后，而不必照搬 DETR。Harness 对照组应包含：单深度输出+原置信度、三个候选但等权监督、三个候选+责任加权、完整排序与多样性正则。观测指标除 KITTI AP3D 外，还要记录候选间深度/yaw 方差、Pearson、Spearman、远距 40m+ 深度 MAE、每目标额外延迟。通过阈值建议在固定 backbone 下 Moderate AP3D 至少提升 1.0 点、Spearman 至少提升 0.10，且三候选推理延迟增加不超过 20%；若候选方差接近零、提升只来自放宽后处理，或 nuScenes MAE 恶化超过 5%，判为失败。

## 优点

- 把“定位多解性”和“选择哪个解”放进同一训练闭环，问题定义清晰。
- 排序质量直接由类别一致的 3D IoU 构造，比深度不确定性更贴近最终 AP。
- 消融覆盖单组件、损失子项、几何多样性、跨数据集深度和计算成本。

## 局限

- 训练依赖 KITTI 小规模数据上的长达 800 epoch 配方及 MixUp3D、DivAlign。
- 每个 RoI 固定三个假设，不能保证适合极密集场景或更多模态的歧义。
- 论文主要面向已知相机几何的自动驾驶数据，开放世界相机内参变化仍需验证。
