## 一句话总结

- **论文链接**：https://openaccess.thecvf.com/content/ICCV2021/html/Xie_Oriented_R-CNN_for_Object_Detection_ICCV_2021_paper.html
- **官方代码**：https://github.com/jbwang1997/OBBDetection

该方法把两阶段旋转检测器的第一阶段改造成 **oriented RPN**：ResNet-FPN 输出五层特征 `{P2,P3,P4,P5,P6}`，每层连接共享结构的 `3×3` 卷积及两个并列 `1×1` 分支。每个位置仅放置长宽比为 `{1:2,1:1,2:1}` 的三个水平锚框，回归分支直接预测六维量 `(δx,δy,δw,δh,δα,δβ)`，解码为外接水平框 `(x,y,w,h)` 与两个中点偏移 `(Δα,Δβ)`，从而直接产生稀疏定向候选框；其参数量约为 RoI Transformer+ 候选生成部分的 `1/3000`、rotated RPN 的 `1/15`。

第二阶段的 **oriented R-CNN head** 接收 `{P2,P3,P4,P5}` 和第一阶段候选框。候选平行四边形先通过延长较短对角线被校正为带方向的矩形，再投影到对应特征层，由 **rotated RoIAlign** 划分为默认 `7×7` 网格并提取旋转对齐特征；特征依次经过 `FC1、FC2`，最后由两个并列全连接层输出 `K+1` 类概率和面向各类别的框回归量。该结构在 DOTA 上以 R-50-FPN 获得 `75.87% mAP、15.1 FPS`，在 HRSC2016 上按 VOC 2012 指标获得 `96.50% mAP`。

论文以 DOTA 和 HRSC2016 为主要验证集，并与 RetinaNet-O、S2ANet、Faster R-CNN-O、RoI Transformer+、R3Det 等基线比较。DOTA 验证集上的候选框数量实验给出了关键取舍：IoU 阈值为 `0.5` 时，top-300、top-1000、top-2000 的召回率分别为 `81.60%`、`92.20%`、`92.80%`；从 2000 减至 1000 仅下降 `0.60` 个百分点，而减至 300 则下降 `11.20` 个百分点，因此测试阶段选用 1000 个候选框。

## 研究背景与问题

旋转检测的效率瓶颈并不完全位于第二阶段，而常来自候选框生成。rotated RPN 在每个位置密集布置不同尺度、比例和角度的锚框，论文举例为 `3尺度×3比例×6角度=54` 个锚框，计算量和显存开销较大。RoI Transformer 则从水平 RoI 学习旋转 RoI，需要依次执行 RPN、RoI Alignment 和回归，候选质量较高但流程昂贵。

作者的核心判断是：若能沿用水平 RPN 的稀疏锚框机制，同时让回归分支直接表达方向，就不必显式枚举角度，也不必增加一个复杂的水平框到旋转框转换阶段。由此，任务被转化为寻找一种数值受约束、容易从水平锚框回归、又可恢复四个旋转顶点的表示。

## 方法总览

**中点偏移表示**以旋转框的最小外接水平矩形为基础。设其中心和尺寸为 `(x,y,w,h)`，旋转框顶点分别由水平矩形上、右、下、左四条边的中点偏移得到：顶部和底部使用互为相反数的水平偏移 `±Δα`，右侧和左侧使用互为相反数的垂直偏移 `±Δβ`。六个参数即可确定四个顶点，且偏移受到外接框宽高的自然约束。

这一表示避免直接回归周期性的角度，也减少了顶点八参数表示的冗余。回归时，中心偏移按锚框宽高归一化，宽高使用对数尺度变化，`Δα、Δβ` 分别按预测框的宽、高归一化。oriented RPN 的分类损失为交叉熵，六维回归使用 Smooth L1；每个小批次默认采样 256 个锚框。

正样本条件为：锚框与任一旋转真值框的外接水平矩形 IoU 高于 `0.7`，或它是某个真值的最高 IoU 锚框且 IoU 高于 `0.3`；IoU 低于 `0.3` 的锚框为负样本，其余忽略。

## 方法详解

推理时，每个 FPN 层先保留 2000 个候选框，并采用 IoU 阈值 `0.8` 的水平 NMS；各层结果合并后按前景分数选择 top-1000 输入第二阶段。分类概率高于 `0.05` 的最终旋转框再按类别执行 polygon NMS，IoU 阈值为 `0.1`。

DOTA 含 2806 张图像、188282 个实例和 15 个类别。原图裁成 `1024×1024` 图块，步长 824，相邻图块重叠 200 像素；训练 12 个 epoch，初始学习率 `0.005`，在第 8、11 个 epoch 除以 10。多尺度方案先按 `{0.5,1.0,1.5}` 缩放，再以步长 524 裁剪。

HRSC2016 含 1061 张船舶图像，训练集 436 张、验证集 181 张；短边缩放至 800，长边不超过 1333，训练 36 个 epoch，并在第 24、33 个 epoch 降低学习率。优化器为 SGD，动量 `0.9`、权重衰减 `0.0001`，训练批量为 2。

## 实验与证据

DOTA 单尺度结果中，R-50-FPN 为 `75.87% mAP`，高于 RoI Transformer+ 的 `74.61%`、S2ANet 的 `74.12%`、Faster R-CNN-O 的 `69.05%`；R-101-FPN 达到 `76.28%`。采用多尺度训练和测试后，R-50-FPN 达到 `80.87%`。

HRSC2016 上，R-50-FPN 获得 `90.40% mAP(07)` 与 `96.50% mAP(12)`，R-101-FPN 为 `90.50%` 与 `97.60%`。统一使用 R-50-FPN、单张 RTX 2080Ti、批量 1 和 `1024×1024` 输入时，该方法为 `15.1 FPS / 75.87 mAP`；RoI Transformer+ 为 `11.3 FPS / 74.61 mAP`，S2ANet 为 `15.3 FPS / 74.12 mAP`，体现了接近单阶段模型的速度和更高精度。

## 对 YOLO-Agent 的启发

对 YOLO-Agent，最值得吸收的不是直接复制两阶段结构，而是把 **中点偏移表示**作为可选旋转框参数化方案。Agent 可在检测头配置中提供 `xywh-angle` 与 `xywh-midpoint-offset` 两种分支，通过数据集验证集自动比较收敛稳定性、旋转 IoU、mAP 与解码耗时。

若系统允许候选区域或二次精修，可引入轻量 oriented proposal head：第一阶段输出六维候选，第二阶段仅对高置信度、密集或细长目标启用 rotated RoIAlign。这样能够把额外计算集中到困难样本，而不是为所有空间位置枚举多个角度锚框。

## 优点

复现实验必须设置三个论文特定对照组：`rotated RPN + 同一检测头`、`RoI Transformer+ + 同一骨干`、`oriented RPN + oriented R-CNN head`。所有组统一使用 R-50-FPN、DOTA 的 `1024×1024` 图块、相同数据增强、训练轮数及 NMS设置，避免把多尺度策略或更强骨干带来的收益归因于候选表示。

## 局限

Harness 同时记录 DOTA mAP、R300/R1000/R2000、FPS、单图延迟、峰值显存和候选生成模块参数量；HRSC2016 分别报告 VOC 2007 与 VOC 2012 mAP。另设候选数量控制实验 `{300,1000,2000}`，检查是否复现 `81.60/92.20/92.80` 的召回趋势，并以 1000 候选测量最终速度—精度平衡。

## 评分

具体失败判据：在同等 R-50-FPN 与单尺度设置下，若 oriented RPN 的 R1000 低于 `92.20%` 超过 1 个百分点，或 DOTA mAP 未达到 `75.87%` 且差距超过 1 个百分点，则判定复现失败；若精度达标但速度不高于 RoI Transformer+ 的 `11.3 FPS`，则判定轻量候选生成目标未实现。还应单独检查密集小目标、极端长宽比目标和跨裁剪边界目标，因为水平 NMS 与外接框匹配可能在这些场景中过早抑制相邻旋转实例。
