## 一句话总结

LSKNet 以串联深度卷积构造多个大感受野候选，再用平均池化、最大池化和逐位置空间掩码选择上下文范围，使桥梁、环岛、球场等遥感目标不必共享同一个固定卷积核尺度。

## 研究背景与问题

遥感图像中的舰船、车辆、桥梁等目标通常尺寸很小，仅凭目标内部纹理容易误判，必须利用远距离环境线索；但不同类别需要的上下文范围又不相同。例如足球场凭边界线即可识别，环岛需要观察周围道路，桥梁则要结合跨水域结构。LSKNet 因此不固定使用单一感受野，而是在每个空间位置动态选择不同尺度的大核特征，以同时满足“需要宽上下文”和“上下文范围随目标变化”两个遥感先验。

LSK 模块的数据流非常明确：输入 \(X\) 先经过串联的深度可分离卷积得到 \(U_1,\ldots,U_N\)，各层采用逐渐增大的卷积核和膨胀率；每个 \(U_i\) 再经独立的 \(1\times1\) 卷积形成 \(\tilde U_i\)。这些特征沿通道拼接后，分别执行通道平均池化和最大池化，拼成双通道空间描述，经卷积映射为 \(N\) 张注意力图并通过 Sigmoid 得到空间选择掩码。各尺度特征按掩码逐像素加权求和，再经卷积生成注意力特征 \(S\)，最终输出为 \(Y=X\odot S\)。

实验覆盖 HRSC2016、DOTA-v1.0 与 FAIR1M-v1.0，并以 Oriented R-CNN 为默认框架，对比 ResNet-18、VAN-B1、MSCAN-S、SKNet-26、ResNeSt-14、SCNet-18、CSPNeXt 以及多种旋转检测方法。LSKNet-S 在 HRSC2016 上取得 VOC 2007/2012 指标下的 90.65%/98.46% mAP，在 DOTA-v1.0 上达到 81.64%，使用 EMA 微调的 LSKNet-S* 达到 81.85%，在 FAIR1M-v1.0 上达到 47.87%。关键消融中，固定理论感受野为 29 时，两层分解 `(5,1)→(7,4)` 获得 80.91% mAP、20.5 FPS，优于单个 `(29,1)` 的 80.66%、18.6 FPS。

## 方法总览

LSK Block 的数据流为：输入依次经过不同核大小和膨胀率的 depth-wise convolution，保留每一级中间特征；各级经 `1×1` 卷积后拼接，通道平均池化与最大池化形成双通道描述，随后卷积和 Sigmoid 产生每个感受野的空间权重，最后完成逐像素加权、卷积映射并与原输入相乘。

## 方法详解

LSKNet 由四阶段重复堆叠 LSKNet Block 构成，每个 Block 包含两个残差子块：负责动态感受野的 **LK Selection** 和负责通道混合、特征细化的 **FFN**。FFN 依次使用全连接层、深度卷积、GELU 和第二个全连接层。LSKNet-T 的阶段通道为 `{32,64,160,256}`，深度为 `{3,3,5,2}`，主干参数量 4.3M；LSKNet-S 为 `{64,128,320,512}` 与 `{2,2,4,2}`，参数量 14.4M。

大核并非直接采用昂贵的单个卷积，而是分解为串联深度卷积。感受野 23 可由 `(5,1)→(7,3)` 获得：64 通道时仅需 11.3K 参数和 11.9G FLOPs，而直接使用 `(23,1)` 需要 40.4K 参数和 42.4G FLOPs。串联结构还会自然产生多个中间感受野，为后续动态选择提供候选，而不是只输出最终最大尺度特征。

空间选择区别于 SKNet 的通道选择：同一张遥感图中，不同位置可能分别出现桥梁、球场和车辆，所需感受野由空间位置决定，通道级统一权重无法表达这种差异。平均池化提供整体响应，最大池化突出显著区域，二者共同生成每个候选大核的逐位置权重。

## 实验与证据

DOTA-v1.0 采用 0.5、1.0、1.5 三尺度训练与测试，并裁剪为重叠 500 像素的 \(1024\times1024\) 图块；FAIR1M-v1.0 使用相同策略。HRSC2016 将长边缩放至 800。主干先在 ImageNet-1K 预训练，消融使用 100 epoch，主结果使用 300 epoch；检测阶段采用 AdamW，HRSC2016 训练 36 epoch，其余数据集训练 12 epoch。

LSKNet-T 在 DOTA-v1.0 上以 21.0M 总参数、124G FLOPs取得 81.37% mAP；LSKNet-S 为 31.0M、161G 和 81.64%，单张 RTX3090 处理 \(1024\times1024\) 图像达到 18.1 FPS。相较 Oriented R-CNN 的 80.87% 和 RVSA 的 81.24%，其精度更高且计算规模明显低于 RVSA 的 114.4M 参数、414G FLOPs。

### 关键消融与分析

- 两层大核分解优于一层和三层：80.91% mAP、20.5 FPS，相比三层方案高 0.14 点且快 1.3 FPS。
- 感受野 23 的 `(5,1)→(7,3)` 配合空间选择达到 81.31%；不做选择仅 80.94%，改用通道选择降至 80.57%。
- 空间选择同时使用最大池化和平均池化得到 81.31%，仅最大池化为 81.23%，仅平均池化为 81.12%。
- 在 Oriented R-CNN、RoI Transformer、S2A-Net、R3Det 中，LSKNet-T 相对 ResNet-18 分别提升 2.04、2.57、3.33、4.23 个 mAP 点，同时主干参数减少 62%、FLOPs 减少 50%。
- 激活分析显示桥梁依赖更大的上下文，足球场依赖最小；网络浅层偏向选择大核，高层更多选择小核，说明宽域信息主要在早期被注入语义特征。

## 对 YOLO-Agent 的启发

可将 LSKNet Block 用作 YOLO-Agent 的可搜索主干候选，而不是无条件替换全部卷积。Harness 至少设置四组控制：原始 YOLO 主干、仅加入相同参数量的大核深度卷积、采用 SKNet 式通道选择、采用完整 LSK 空间选择；检测头、旋转框编码、训练尺度和增强策略必须保持一致。另设“仅替换浅层”和“全阶段替换”，验证论文观察到的浅层大核偏好。

评估应在 DOTA-v1.0 报告旋转框 mAP、桥梁/环岛/足球场分类别 AP、参数量、\(1024^2\) FLOPs、FPS 与峰值显存，并记录大小核掩码差异。具体失败标准：完整 LSK 组若未比等参数大核组提升至少 0.5 mAP，或桥梁 AP 无提升且 FPS 下降超过 10%，则判定动态空间选择没有带来足以覆盖部署成本的收益；若仅足球场等纹理明确类别提升，则应拒绝“宽上下文普遍有效”的结论。

## 优点

- 从遥感目标的上下文差异出发设计主干，而非继续堆叠旋转框技巧。
- 大核分解同时降低计算量并保留多个候选感受野。
- 空间级动态选择与多目标、高分辨率遥感图像高度匹配。
- 在多种一阶段、两阶段检测框架中均有稳定增益。

## 局限

- 主结果依赖 ImageNet 300 epoch 预训练和 DOTA 多尺度测试，训练成本不低。
- 论文主要验证旋转检测，未系统讨论水平框、密集预测或端侧部署。
- 最大、平均池化产生的选择权重较粗，复杂遮挡下不一定精确对应真实上下文。
- 主要报告整体速度，缺少不同硬件及算子实现下的延迟分析。

## 评分

该工作的关键并非“卷积核越大越好”，而是先高效构造多个宽域候选，再按空间位置选择。对遥感检测代理而言，搜索空间应包含感受野、分解层数、膨胀率、插入阶段和选择方式，并以类别级上下文收益约束搜索，而不能只优化总体 mAP。
