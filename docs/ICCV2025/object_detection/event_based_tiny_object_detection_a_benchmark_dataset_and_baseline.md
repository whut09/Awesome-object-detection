# Event-based Tiny Object Detection: A Benchmark Dataset and Baseline

**论文**：[CVF 官方论文页面](https://openaccess.thecvf.com/content/ICCV2025/html/Chen_Event-based_Tiny_Object_Detection_A_Benchmark_Dataset_and_Baseline_ICCV_2025_paper.html)  
**代码**：[官方代码与数据集](https://github.com/ChenYichen9527/Ev-UAV)  
**发表**：ICCV 2025  
**类别**：Event-based Detection, Tiny UAV, Point Cloud Segmentation

## 一句话总结

论文用 EV-UAV 建立事件级微小无人机基准，并以 EV-SpSegNet 直接在 XYT 稀疏点云中提取连续运动曲线，再用 Spatiotemporal Correlation Loss 强化成簇目标事件、抑制孤立噪声。

## 研究背景与问题

普通 RGB 相机受 30–60 Hz 帧率、动态范围和曝光限制，高速或极端光照下的小型无人机容易模糊、欠曝或过曝。事件相机以微秒级时间戳记录亮度变化，天然保留运动轨迹，但现有事件检测数据集目标偏大、场景单一，多数方法还会把事件压成帧，损失时序连续性并重新引入稠密背景计算。本文要解决两个相互关联的问题：缺少真正面向微小反无人机的事件级基准，以及缺少能直接利用“运动目标在时空点云中形成连续曲线”这一结构的高效模型。

## 方法总览

EV-UAV 由 DAVIS346 采集，分辨率 346×260，包含 147 段序列和超过 2030 万目标事件；训练/验证/测试为 99/24/24 段。作者先按时间窗积累事件帧并标二维框，再沿时间维扩展为 XYT 三维框，框内事件被标为目标，从而得到事件级分割监督。EV-SpSegNet 采用对称 U 形稀疏网络：体素化后进入 Grouped Dilated Sparse Convolution Attention（GDSCA），其中 GDSC 按通道分组使用不同膨胀率提取多尺度局部时序结构，Sparse-SE 融合各组，Patch Attention 在下采样后的大块之间交换全局上下文。STC Loss 则以邻域预测置信度构造时空相关权重，对连续轨迹事件加大正类损失，对孤立高置信噪声加强负类约束。

## 方法详解

输入是连续 8 秒事件，体素大小为 1 像素×1 像素×1 ms。GDSC 的四个分支使用不同膨胀率，使有限稀疏卷积既看到局部曲线切片，又覆盖更长时间跨度；随后 Patch Attention 不是对全部事件做二次复杂度注意力，而是在子区域级建模全局关系。编码器逐层压缩，解码器通过跳连恢复事件级预测，最终线性层与 Sigmoid 输出每个事件属于目标的概率。

STC Loss 在每个事件周围取 `k×k×τ` 邻域，累计支持事件置信度形成权重，指数 γ 控制强调程度。论文设置 k=3、τ=5、γ=2。与逐点 BCE 不同，它把“邻域中是否存在连续支持”直接写进损失：轨迹内部漏分会付出更高代价，孤立噪声的假阳性也会被更强惩罚，因此监督对象从单点类别变为局部运动结构。

## 实验与证据

EV-UAV 平均目标仅 6.8×5.4 像素，极微小目标约占 45%，覆盖普通光、强光、弱光，多目标、复杂建筑/山林/水面背景。评测使用事件分割 IoU、ACC，以及定位 Pd 和 Fa。EV-SpSegNet 达到 55.18 IoU、65.02 ACC、77.53 Pd、1.63×10^-4 Fa，4.0M 参数；处理 8 秒事件仅 35.9 ms。对比中 COSeg 为 51.89 IoU、60.93 ACC、71.32 Pd，帧式与事件帧检测器的误报明显更高。消融从 51.36 IoU 基线出发：仅 GDSC 为 52.73；GDSC+Patch Attention 为 53.62；再加 STC 达 55.18。GDSC 分支数 4 最优，膨胀率组合 (1,2,3,4) 优于更稀疏的大间隔组合。STC 加到 KPConv、RandLA-Net、COSeg 上均提升 IoU 并降低 Fa，例如 KPConv 的 IoU 提升 2.13，Fa 从 16.32 降到 10.60。

## 对 YOLO-Agent 的启发

这项工作不应被简单改写成 RGB 小目标 neck。适合的 Agent 任务是事件点云分割或“事件分割→时间窗框化”的两阶段管线，并保留运动连续性指标。

**机制特定 Harness**：**对照组**沿用相同事件窗口与点采样，比较事件帧 YOLO、稀疏 U-Net+BCE、EV-SpSegNet+BCE、EV-SpSegNet+STC，并增加打乱时间戳后训练的 STC 反事实组。**指标**除论文的 IoU、ACC、Pd、Fa 外，按目标像素面积、运动速度、光照、噪声比例、目标数量和轨迹交叉程度统计轨迹断裂数、真轨迹保留长度、孤立误报簇及 8 秒序列端到端延迟。**失败判断**围绕时空连续性制定：STC 相对 BCE 的轨迹断裂降幅小于 10%，或 Fa 增幅超过 5%，或打乱时间戳后 IoU 下降不足 2 个百分点，任一结果都说明 STC 没有提供可信的运动约束；慢速桶 Pd 再低于全量 Pd 的 80% 时，部署方案还必须补充其他模态。

## 优点

- 数据集目标尺度、光照与事件级标注均直指反无人机场景。
- 模型直接处理稀疏事件，避免事件帧转换的时序损失与冗余计算。
- STC 在多种点云网络上有效，说明损失具有一定通用性。

## 局限

- 事件相机无法稳定观测静止或极慢目标，论文明确指出需要 RGB 互补。
- 由二维框扩展到三维事件框是近似标注，复杂交叉轨迹可能混入背景事件。
- 任务输出首先是事件分割，转成常规检测框仍需额外聚类与时序规则。

## 评分

**9.0/10**。数据、结构和损失围绕事件连续曲线形成闭环，速度与误报优势突出；主要风险来自事件传感器固有的静止目标盲区和近似标注。
