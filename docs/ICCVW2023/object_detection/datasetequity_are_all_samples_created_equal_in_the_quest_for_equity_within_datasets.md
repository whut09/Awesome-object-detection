# DatasetEquity: Are All Samples Created Equal? In The Quest For Equity Within Datasets

**论文**：[官方论文](https://doi.org/10.1109/ICCVW60793.2023.00476)  
**代码**：论文未给出可确认的独立官方仓库

## 一句话总结

DatasetEquity 讨论数据集中样本贡献的不均衡：相同数量的样本不代表相同训练价值，删选数据时既要考虑性能贡献，也要防止某些类别或子群被进一步边缘化。

## 研究背景与问题

常见数据裁剪按随机、类别频率或模型置信度选样，但这些规则可能保留大量重复样本，同时删除稀有场景。论文把 equity 引入样本价值分析，关注同一预算下不同子集对总体性能和群体差距的影响。

## 方法总览

流程先为样本计算价值或代表性排序，再构造等规模的高分、低分、随机和类别均衡子集。每个子集从相同初始化训练，比较总体检测质量、类别覆盖和子群差距；因此判断对象不是单张图“难不难”，而是它对数据组合的边际贡献。

## 方法详解

### 1. 固定模型、训练步数和随机种子，避免把更长训练误当成高价值样本效果。

### 2. 按照样本评分生成多个预算比例的子集，同时保留随机与类别均衡对照。

### 3. 分别重训并观察总体指标以及类别/群体指标，检查收益是否只来自头部类别。

### 4. 比较不同种子下的样本排序稳定性；不稳定排名不能用于长期数据治理。

## 实验与证据

论文的核心表格结论是：样本价值存在明显差异，等规模子集可得到不同模型表现；按价值选择与随机、类别均衡选择的相对次序会随评价目标改变。高价值子集提升总体指标时，也可能扩大部分群体差距，因此“更高平均性能”不能直接等同于“更公平的数据”。本笔记不补造无法从官方页面可靠核对的具体百分数。

## 对 YOLO-Agent 的启发

设置 subset_ratio∈{0.1,0.25,0.5}、selector∈{value,random,class_balanced}、group_metric。每种子集至少跑三种子，保存总体 AP、逐类 AP、最差群体 AP 和排名相关系数。若 value 子集只提高头部类、最差群体持续下降，或跨种子排序相关性接近零，则拒绝该选样器。

## 优点

把样本裁剪从单一平均性能扩展到类别与群体后果。

## 局限

样本价值依赖模型和评价目标，换检测器后原排序未必可迁移。
