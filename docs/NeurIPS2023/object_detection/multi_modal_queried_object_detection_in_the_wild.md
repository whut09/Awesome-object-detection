# Multi-modal Queried Object Detection in the Wild

## 一句话总结

该工作在冻结的 GLIP 或 GroundingDINO 文本查询检测器中插入 **Gated Class-scalable Perceiver（GCP）**，用类别文本与少量视觉样例共同定义目标，并通过 **vision-conditioned masked language prediction** 克服模型只依赖文本的学习惯性，以一次 Objects365 调制训练换取显著的免微调与少样本增益。

## 研究背景与问题

现有开放词汇检测器把检测改写为区域—词语匹配，文本具有大词表泛化能力，却难以表达同名异义、外观纹理和细粒度类别；例如 GLIP 在 Aquarium 上加入人工类别描述后，AP 仅由 17.7% 升至 18.4%。纯视觉样例虽包含更具体的外观信息，但传统双分支少样本检测通常需要重新训练或深度耦合，容易遗忘基础模型已学习的开放概念。

作者因此研究一个不同于标准闭集检测的问题：推理时，用户可以提供类别名称、每类若干带框视觉样例，或二者同时提供；模型需要在**不做下游微调**的条件下检测任意粒度的新类别。核心约束是新增机制必须类别可扩展、能融合视觉细节，同时保留原语言检测器的开放集能力。

## 方法总览

MQ-Det 不替换检测主干，而是在冻结文本编码器的最后六层之间加入 GCP。目标图像进入原图像编码器；类别文本进入原文本编码器；每类视觉样例经同一图像编码器与 RoI Pooling 得到查询特征。GCP 先让视觉查询关注目标图像，再让对应类别 token 关注这些视觉查询，最后通过内容相关的门控残差修正文本表示。检测头仍计算区域—词语相似度，并沿用原模型的 phrase grounding loss 与定位损失。

训练与推理边界非常清晰：Objects365 上仅训练新加入的 GCP，整个预训练检测器保持冻结；推理时无需更新参数，只需替换文本和视觉查询。论文页面：[NeurIPS 2023 官方页面](https://proceedings.neurips.cc/paper_files/paper/2023/hash/0e3af444e7d82d29871804de476d1fbe-Abstract-Conference.html)。作者提供的官方代码：[YifanXu74/MQ-Det](https://github.com/YifanXu74/MQ-Det)。

## 方法详解

### 类别级多模态数据流

对类别 \(i\) 的第 \(j\) 个标注样例，模型从扩张后的框中提取视觉查询：

\[
v_i^{(j)}=\operatorname{RoIPool}(\phi_I(I),\gamma b)
\]

其中区域面积按 \(1.5^2\) 扩张，以纳入上下文。每类离线查询库最多保存 \(K=5000\) 个特征；调制训练的每次前向随机抽取 \(k=5\) 个，模拟真实低样本输入。

GCP 对每个类别独立计算：

\[
\hat v_i=\operatorname{X\!-\!MHA}(v_i,\mathbf I),\qquad
\tilde v_i=\operatorname{X\!-\!MHA}(t_i,\hat v_i)
\]

第一层交叉注意力使样例查询结合当前目标图像内容，第二层用类别 token 汇聚多视角视觉证据。类别级 attention mask 阻止某一类别的文本访问其他类别的样例，因此模块没有类别专属参数，可扩展到任意类别数和任意推理查询数。

融合结果为：

\[
\tilde t_i=t_i+\tanh(\operatorname{gate}(\tilde v_i))\cdot\tilde v_i
\]

`gate` 是逐层设置的三层 MLP，将特征逐步压缩成一个标量，用于估计该类视觉样例的可靠性。门值初始化为零，使初始输出严格贴近冻结文本编码器，避免随机视觉分支立即破坏原语义空间。

### 视觉条件文本遮蔽

若直接训练残差式 GCP，冻结模型会停留在原有“区域对齐文本”的局部最优。作者以 40% 概率把当前图像中真实类别对应的文本 token 替换为 `[MASK]`，迫使检测预测依赖视觉查询。这里并非额外训练一个 token 重建器；监督仍来自目标检测的 grounding/classification 与 box localization，只是分类语义必须经过视觉样例补全。

## 实验与证据

调制训练使用 Objects365：66 万张图像、365 类、约 3000 万个框，只训练一轮、使用 8 张 V100。评估覆盖 LVIS 的 1203 类长尾检测，以及真实场景集合 ODinW-13/35；免微调协议从下游训练集为每类抽取 5 个实例，仅作为查询，不更新模型。

- **LVIS Val v1.0：**MQ-GLIP-L 从 GLIP-L 的 26.9 AP 提升至 **34.7 AP（+7.8）**；稀有类 APr 从 17.1 升至 26.9。
- **LVIS MiniVal：**GLIP-T 为 26.0 AP，MQ-GLIP-T 达 **30.4 AP**；GroundingDINO-T 为 25.7，MQ-GroundingDINO-T 达 30.2。
- **ODinW-13：**免微调由 GLIP-T 的 41.9 提升至 **45.6 APavg**；3-shot 由 50.7 提升至 **57.0（+6.3）**。
- **ODinW-35：**MQ-GLIP-T 的 3-shot APavg 为 **43.0**，高于 GLIP-T 的 38.9。
- **显式新类划分：**排除 Objects365 中出现的类别后，MQ-GLIP-L 在 LVIS novel 类上由 35.4 提升至 **41.7 AP**，说明收益并非只来自调制类别记忆。

关键受控消融支持其机制解释：取消遮蔽时 LVIS AP 从 30.4 降至 **26.9**，80% 遮蔽也仅有 28.9；移除门控降至 **27.1**，共享标量门控为 29.3，线性门控为 30.0；训练整个检测器反而只有 **24.4 AP**，仅解冻文本编码器为 26.6，均明显弱于冻结主模型、只训练 GCP 的方案。

## 对 YOLO-Agent 的启发

### 专属 Harness

可将 MQ-Det 设计成 YOLO-Agent 的“临时类别注册器”：Agent 接收自然语言类别与带框样例，调用冻结 YOLO 视觉编码器建立类别查询库，再以类别级交叉注意力和零初始化门控调制开放词汇分类嵌入；新增类别不触发主模型训练，查询质量较差时门控应自动降低视觉残差。

验证必须设置四个精确控制组：**A** 为文本查询 YOLO-Agent；**B** 为仅视觉查询；**C** 为文本与视觉特征直接平均；**D** 为 GCP 式两级交叉注意力、三层条件门控和 40% 文本遮蔽。统一使用每类 5 个样例，在 LVIS MiniVal 和 ODinW-13 免微调协议下报告 AP、APr、APavg、APmid、每类查询编码时间与单图延迟；另在排除训练词表的新类集合上报告 APnovel。

可证伪标准为：若 D 相对 A 在 LVIS MiniVal 的 AP 增益低于 **3 点**，或在 ODinW-13 的 APavg 不优于 C，或加入视觉查询后无样例类别的 AP 下降超过 **1 点**，则“门控视觉查询能够在保持开放词汇能力的同时提供细粒度增益”这一迁移假设失败。还应复现 0%、40%、80% 遮蔽率，以确认改进来自视觉依赖建立，而非新增参数量。

## 优点

- 直接复用成熟语言查询检测器，只增加可插拔 GCP，训练成本远低于重新进行大规模视觉—语言预训练。
- 类别级共享参数与 attention mask 同时实现类别隔离和类别数扩展，推理查询数不受训练时 \(k=5\) 限制。
- 冻结基础模型并采用零初始化门控，有明确的抗遗忘设计，而非仅凭经验减少学习率。
- 实验同时覆盖免微调、少样本、满样本和严格排除 Objects365 类别的新类协议，证据链较完整。

## 局限

视觉样例的收益会随监督数据增加而减弱：ODinW-13 满样本下 MQ-GLIP-T 仅从 61.9 提升至 62.5 AP。查询质量仍是外部变量，严重遮挡、错误框或背景占比过高的样例可能无法提供有效细节。方法还需要为每类保存和编码视觉查询，类别极多时存在存储与调制开销；论文也未验证实例分割等其他密集预测任务，并指出开放词表能力可能被滥用于定向监控。

## 评分

**8.8/10。** 方法结构克制，训练—推理边界明确，消融能够分别验证遮蔽、门控和冻结策略；尤其是 +7.8 LVIS AP 与显式 novel-class 结果具有说服力。扣分主要来自对样例质量的依赖、满样本收益有限，以及尚未系统研究错误查询、超大类别库和其他密集预测任务。
