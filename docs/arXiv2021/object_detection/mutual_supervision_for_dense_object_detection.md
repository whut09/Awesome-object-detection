# Mutual Supervision for Dense Object Detection

**论文**：[官方论文页面](https://arxiv.org/abs/2109.05986)  
**代码**：论文未提供官方代码  
**发表**：arXiv 2021

## 一句话总结

Mutual Supervision（MuSu）在每个真值的自适应候选袋中，用回归 IoU 排序监督分类头、用分类置信度排序监督回归头，再把排序转成软权重，从训练阶段约束两条分支对同一批高质量位置达成一致。

## 研究背景与问题

密集检测最终用分类分数排序并执行 NMS，但分类头的正样本常由几何中心或固定 IoU 规则确定，未必对应回归最准确的位置。反过来，回归头也可能在分类不可靠的位置上消耗大量梯度。已有方法常把分类分数与 IoU 合成一个统一质量，但 MuSu 认为两条分支的监督不必完全相同：分类应更相信定位好的候选，回归应更相信类别判断稳定的候选，并允许两者拥有不同的有效正样本数量。

## 方法总览

MuSu 不改推理结构，而是在训练时执行三步：先用分类分数和 IoU 的联合似然为每个实例建立候选袋；再分别计算分类与回归的互监督准则并排序；最后以负指数函数把名次转为软权重，叠加到 focal loss 与 GIoU loss 上。它是样本分配与损失加权的上层框架，而不是新的基础检测损失。

算法输入是跨所有金字塔层的 anchor 集合与真值集合。对每个真值完成候选过滤、两套排序和权重计算后，再汇总整张图的分类与回归损失。背景位置仍承担负类 focal loss；候选位置除被分配的类别外，其他类别也保留负类惩罚。因此 MuSu 并不是把候选袋外全部梯度删除，而是精细控制“哪个实例监督这个位置”以及“该位置对两个 head 各贡献多少”。

## 方法详解

### 1. 自适应候选袋

对真值 `j` 和位于框内的 anchor `i`，记类别分数为 `p_i`，与真值的 IoU 为 `IoU_i`，缩放后的定位质量为 `q_i=IoU_i^θ`。联合似然为 `P_i=p_i q_i`，论文取 `θ=4`。候选阈值是 `t=b·max_i P_i`，满足 `P_i>t` 的位置进入 `C_j`，默认 `b=0.1`。若一个 anchor 同时落入多个框，只保留 IoU 最大的真值，避免候选袋冲突。分类与定位越错位，最大联合似然越低，袋子会自动扩大，从而给困难实例更多候选。

### 2. 两条分支互相排序

直接用 `q_i` 监督分类、用 `p_i` 监督回归会在训练早期受噪声影响，因此 MuSu 加入本分支输出作为正则：`v_i^cls = q_i · p_i^α`，`v_i^reg = p_i · q_i^α`。`α∈[0,1]` 控制“纯互监督”和“共同联合质量”之间的程度。`α=0` 是完全由对方分支决定，`α=1` 时两条分支使用相同联合准则；实验最优为 `α=1/3`。在每个 `C_j` 内分别按 `v^cls`、`v^reg` 降序得到从 0 开始的排名 `R_i^cls、R_i^reg`。

### 3. 排名到软权重

权重定义为 `w_i^cls=exp(-R_i^cls/τ_cls)`、`w_i^reg=exp(-R_i^reg/τ_reg)`。温度越大，获得有效梯度的候选越多。论文把温度随候选袋大小自适应设置，并发现 `τ_cls:τ_reg=2:1` 较好，即分类可以吸收更多位置，回归集中在更少的高置信候选。通用加权形式为 `L=(1/N)Σ_i w_i l_i`，其中 `N=Σ_i w_i`，`l_i` 可分别取 focal loss 和 GIoU loss；检测总损失为 `L_cls+L_reg`。

用排名而不是准则绝对值还有一个作用：不同实例的分类分数和 IoU 标度可能差异很大，但袋内名次只表达相对优劣，不会让容易实例凭借整体更高分数完全压过困难实例。负指数映射又比 top-k 硬切分平滑，第一名权重为 1，之后按温度逐步衰减；分类和回归采用不同温度，正是 MuSu 区别于单一联合质量分配的关键。

## 实验与证据

实验使用 MS COCO train2017 训练、5K minival 消融，并提交 test-dev；主要基线是 FCOS 与加入 center sampling 等技巧的 FCOS++。ResNet-50、90K iteration 下，FCOS 为 36.5 AP，FCOS++ 为 38.6。仅由分类或仅由 IoU 单向决定两头权重分别为 38.3 和 31.8 AP；无正则的纯互监督 `α=0` 为 38.5，而 `α=1/3` 达 40.6 AP、44.3 AP75，说明对方分支监督必须保留本分支稳定项。

候选阈值 `b=0.1` 得 40.6 AP，优于 0.2 的 40.3 和 0.05 的 40.4；固定温度不及随 `|C_j|` 自适应。软权重为 40.6 AP，硬阈值版本低 0.6 AP。每位置增加 anchor 时，`α=1` 从 40.4 降至 39.9，而 `α=1/3` 在 3 个 anchor 时升至 40.9，表明分支特异排序能利用额外候选。COCO test-dev 上，MuSu 的 ResNet-101 为 44.8 AP、49.1 AP75，与 PAA 的 44.8 AP 相当且 AP75 更高；ResNet-101-DCN 为 47.4 AP、51.8 AP75。

训练配置也保持常规：SGD 学习率 0.01、momentum 0.9、weight decay 0.0001，总 batch 16；90K 日程在 60K、80K 衰减，推理分数阈值 0.05、NMS 阈值 0.6。由此可以把性能变化主要归因于样本监督，而不是额外推理技巧。论文附录报告多 anchor 只带来轻微速度下降，MuSu 本身在测试时被完全移除。

## 对 YOLO-Agent 的启发

接入点应放在 assigner 输出之后、分类与框损失聚合之前：为每个 GT 建候选袋，用当前类别分数与 CIoU/IoU 计算两套排名权重，而不是让分类和回归共享一组 foreground mask。对照组设置为原 assigner、统一联合质量权重、MuSu 硬权重、MuSu 软权重，并固定候选生成半径。关键指标除 COCO AP/AP75 外，应记录分类与回归有效样本数比、每 GT 候选袋大小和正样本梯度方差。若 AP75 未提升 0.8 点，或 `τ_cls:τ_reg=2:1` 下回归有效样本中位数低于 2、训练出现连续 500 iteration 的 box loss 上升，则视为互监督过度收缩并回退到统一质量分配。

## 优点

- 明确允许分类和回归拥有不同样本排序与不同监督密度。
- 只改变训练，推理阶段没有额外分支或后处理成本。
- 对软目标、温度比例、多 anchor 和单向/互向监督给出了系统消融。

## 局限

- 候选质量依赖训练中的即时预测，早期噪声仍需阈值、指数和正则共同抑制。
- `θ、b、α、τ` 形成一组耦合超参数，迁移到不同 assigner 时调参成本不低。
- 论文只在 FCOS 风格实现和 COCO 上充分验证，其他检测范式的通用性仍需复现。

## 评分

**8.7/10**：把“分类监督来自定位、回归监督来自分类”落实成可计算的双排序软分配，机制清楚且消融充分，但超参数和早期预测噪声使工程复现比表面上更复杂。
